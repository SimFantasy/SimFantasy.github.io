Pinia使用小结

Simz
2022-04-29
pinia vuex vue

大菠萝Pinia，号称vux5，全新的vue状态管理库。

个人感受，vuex4.x里面的概念太多了，本来vue就喜欢造概念，一堆一堆学都学不过来。vue4.x里更是繁琐的很，components要DIspatch给Actions，然后Actions要commit给Mutations，当然也可以从commponents给到Mutations，然后Mutaiions要Mutate给State。

但凡可以 get和set就别造新词了吧，真的记不住~ 搞得我看了好几遍，现在还是没啥概念。

Pinia就好很多 Actions既可以同步也可以异步，没分开。数据传递也很简单。看着教程就很容易的上手。

安装

1npm i pinia -D

导入、实例化、挂载

main.js

 1import { createApp } from 'vue'
 2import App from './App.vue'
 3import { createPinia } from 'pinia'
 4// 创建Pinia实例
 5const pinia = createPinia()
 6// 实例化 Vue
 7const app = createApp(App)
 8
 9// 挂载pinia
10app.use(pinia)
11app.mount('#app')

创建store

src/store/index.js

  1. 可以有多个容器，通过容器ID进行区分。
  2. 定义store有函数和对象两种写法。
 1import { defineStore } from 'pinia'
 2
 3// 参数1：容器的ID，必须唯一 将来pinia会把所有的容易挂载到根容器
 4// 参数2：选项对象
 5// 返回值是一个函数 调用得到  对外部暴露一个 use 方法，该方法会导出我们定义的 state
 6export const userMainStore = defineStore('main', {
 7  // 类似于组件的data 用来存储全局状态
 8  state: () => {
 9    return {
10      counter: 100,
11      msg: 'Hello Pinia'
12    }
13  },
14  getters: {
15    // getters 相当于computed，对state中的数据进行加工
16    // 如果使用箭头函数会导致 this 指向有问题
17    // 可以在函数的第一个参数中拿到 state
18    getCounter (state) {
19      return state.counter * 2
20    }
21  },
22  actions: {
23    // actions 相当于methods用来修改 state
24    add () {
25      this.counter++
26    },
27    sub () {
28      this.counter--
29    },
30    changeMsg () {
31      this.msg = 'Hello Sim'
32    }
33  }
34})

src/components/HelloWorld.vue

导入store中的userMainStore，可以在组件中调用，store中定义在Actions中的方法。使用公共方法对state中的数据做操作。

 1<template>
 2  <div>HelloWorld: {{store.counter}}</div>
 3  <button @click="add">+1</button>
 4  <button @click="sub">-1</button>
 5  <button @click="addTen">+10</button>
 6  <br>
 7  <hr>
 8  <br>
 9  <div>HelloWorld: {{store.msg}}</div>
10  <button @click="changeActionsMsg">change Actions msg</button>
11  <button @click="changeMsg">change msg</button>
12  <br>
13  <hr>
14  <div>HelloWorld: {{ store.getCounter}}</div>
15</template>
16
17<script setup>
18import { userMainStore } from '../store/index'
19
20const store = userMainStore()
21
22const add = () => {
23  store.add()
24}
25const sub = () => {
26  store.sub()
27}
28
29const addTen = () => {
30  store.$patch({
31    counter: store.counter + 10
32  })
33}
34
35const changeActionsMsg = () => {
36  store.changeMsg()
37}
38
39const changeMsg = () => {
40  store.$patch((state) => {
41    state.msg = 'Hello Simz'
42  })
43}
44</script>

src/components/Home.vue

1.直接从 store中解构的对象，不具备响应式，使用storeToRefs解构后的具有响应式

 1<template>
 2  <div>Home:{{store.counter}}</div>
 3  <div>Home:{{store.msg}}</div>
 4  <br>
 5  <hr>
 6  <br>
 7  <div>{{counter}}</div>
 8  <div>{{msg}}</div>
 9</template>
10
11<script setup>
12import { userMainStore } from '../store/index'
13import { storeToRefs } from 'pinia'
14
15// 使用storeToRefs解构后具有响应式
16const store = userMainStore()
17const { counter, msg } = storeToRefs(store)
18</script>
