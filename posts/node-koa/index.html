Koa Node

koa 是由 Express 原班人马打造的，致力于成为一个更小、更富有表现力、更健壮的 Web 框架。使用 koa 编写 web 应用，通过组合不同的 generator，可以免除重复繁琐的回调函数嵌套，并极大地提升错误处理的效率。koa 不在内核方法中绑定任何中间件，它仅仅提供了一个轻量优雅的函数库，使得编写 Web 应用变得得心应手。

安装：

1pnpm add koa

koa 最基础的web服务

示例：

  • 在服务端打印hello world
  • 在客户端响应hello koa
 1const Koa = require('koa')
 2const app = new Koa()
 3
 4app.use((ctx, next) => {
 5	console.log('hello world')
 6	next()
 7})
 8
 9app.use((ctx, next) => {
10	ctx.body = 'hello koa'
11})
12
13app.listen(3000, () => {
14	console.log('server is running at http://localhost:3000')
15})

Koa注册的中间件提供两个参数：

  • ctx： 上下文Context对象。
    • ctx表示依次请求的上下文对象。
    • koa将req和res封装到了ctx中，ctx.request和ctx.response。
  • next本质上是一个dispatch，类似于express中的next，但是有区别。
    • koa的next返回一个promise，而express的next仅为一个函数。
    • koa的next不能传递值，但express的next可以。

中间件

koa通过创建app对象，注册中间件只能通过use方法，并没有提供类似express的methods方法来注册中间，也没有提供path中间件来匹配路径。

在koa中我们使用第三方路由中间件来实现。

目前主要有koa-router和@koa/router，koa-router已经在2016年后未在进行维护，所以我们使用@koa/router，使用方式基本和koa-router差不多，和express中的router使用也类似。

安装：

1# koa-router
2pnpm add koa-router
3
4# @koa/router
5pnpm add @koa/router

router使用基础示例：

 1const Koa = require('koa')
 2const Router = require('@koa/router')
 3
 4// 创建Koa实例作为app
 5const app = new Koa()
 6// 创建路由实例
 7const router = new Router()
 8
 9router.get('/', (ctx, next) => {
10	ctx.body = 'hello world'
11})
12
13router.post('/login', (ctx, next) => {
14	ctx.body = 'login success'
15})
16
17// 将路由实例挂载到app上
18app.use(router.routes())
19// 允许使用get、post等方法,其他未使用的方法会返回405 Method Not Allowed
20app.use(router.allowedMethods())
21
22// 监听3000端口
23app.listen(3000, () => {
24	console.log('server is running at http://localhost:3000')
25})

allowedMethods用于判断某一个method是否支持

  • 如果我们请求 get、post，那么是正常的请求，因为我们有实现get、post
  • 如果我们请求 put、delete、patch，那么就自动报错：Method Not Allowed，状态码：405
  • 如果我们请求 link、copy、lock，那么就自动报错：Not Implemented，状态码：501

@koa/router也支持类似express router中类似的route的用法。

 1router
 2  .get('/', (ctx, next) => {
 3    ctx.body = 'Hello World!';
 4  })
 5  .post('/users', (ctx, next) => {
 6    // ...
 7  })
 8  .put('/users/:id', (ctx, next) => {
 9    // ...
10  })
11  .del('/users/:id', (ctx, next) => {
12    // ...
13  })
14  .all('/users/:id', (ctx, next) => {
15    // ...
16  });

router.all()可以用于匹配所有的方法。

另外，也支持多个中间件。

 1router.get(
 2  '/users/:id',
 3  (ctx, next) => {
 4    return User.findOne(ctx.params.id).then(function(user) {
 5      ctx.user = user;
 6      next();
 7    });
 8  },
 9  ctx => {
10    console.log(ctx.user);
11    // => { id: 17, name: "Alex" }
12  }
13)

Request 数据解析

koa中路由传参和express类似，但koa依赖中间件来get请求时处理json参数和post请求时处理body中参数。

Params

 1const Koa = require('koa')
 2const Router = require('@koa/router')
 3
 4const app = new Koa()
 5const router = new Router()
 6
 7// http://localhost:3000/posts/1
 8router.get('/posts/:id', (ctx, next) => {
 9	const { id } = ctx.params
10	ctx.body = `post ${id}`
11})
12
13app.use(router.routes())
14
15app.listen(3000, () => {
16	console.log('server is running at http://localhost:3000')
17})

Query

koa中直接将querystring解析成了对象，不需要使用其他中间件处理。

 1const Koa = require('koa')
 2const Router = require('@koa/router')
 3
 4const app = new Koa()
 5const router = new Router()
 6
 7// http://localhost:3000/posts?limit=10&offset=0
 8router.get('/posts', (ctx, next) => {
 9	const query = ctx.query
10	ctx.body = query
11})
12
13app.use(router.routes())
14
15app.listen(3000, () => {
16	console.log('server is running at http://localhost:3000')
17})
18// {
19//   "limit": "10",
20//   "offset": "0"
21// }

同样可以使用koa-bodyparser来解析对应的body数据，代码同json，仅提交请求的方式不一样，不做重复说明。

文件上传

文件上传也是使用@koa/multer这个第三方中间件，操作方式同express类似。

单文件上传：

 1const path = require('path')
 2const Koa = require('koa')
 3const Router = require('@koa/router')
 4const multer = require('@koa/multer')
 5const dayjs = require('dayjs')
 6const {mkdirp} = require('mkdirp')
 7
 8const app = new Koa()
 9const router = new Router()
10
11// 直接上传到upload目录下
12// const storage = multer.diskStorage({
13// 	destination: (req, file, cb) => {
14// 		cb(null, './uploads')
15// 	},
16// 	filename: (ctx, file, cb) => {
17// 		cb(null, Date.now() + path.extname(file.originalname))
18// 	}
19// })
20
21// 上传到年月目录下
22const storage = multer.diskStorage({
23	destination: async (req, file, cb) => {
24		// 生成日期目录所需的年月
25		const yearMonth = dayjs(new Date()).format('YYYYMM')
26		// 生成目录路径
27		const dir = path.join(__dirname, `./uploads/${yearMonth}`)
28		// 生成目录
29		await mkdirp(dir)
30		// 将目录路径传入 callback，在传入目录之前目录必须存在
31		cb(null, dir)
32	},
33	filename: (ctx, file, cb) => {
34		cb(null, Date.now() + path.extname(file.originalname))
35	}
36})
37
38const upload = multer({ storage })
39
40// http://localhost:3000/upload
41router.post('/upload', upload.single('avatar'), (ctx, next) => {
42  console.log(ctx.request.file)
43  const imagedir = ctx.file.destination.split('\\').pop()
44  const imagename = ctx.file.filename
45  const imagepath = `/uploads/${imagedir}/${imagename}`
46	ctx.body = imagepath
47})
48
49app.use(router.routes())
50
51app.listen(3000, () => {
52	console.log('server is running at http://localhost:3000')
53})

此处ctx.file 等同于ctx.request.file， koa把ctx.request中一些方法放到外面一层而已。类似的还有ctx.body等同于ctx.request.body

Response 响应数据

body

koa中主要使用ctx.body将相依主体输出。body类似与express中的send。

  • body支持数据类型：string、buffer、stream、object、array、null，如返回null则不输出任何内容。
  • ctx.body等同于ctx.response.body

status

status和express中类似，可以直接设置响应状态。

  • 使用：ctx.status = 200

  • ctx.status等同于ctx.response.status

静态服务

koa中没有内置相关的静态服务功能，需要使用第三方中间件koa-static。

安装：

1pnpm add koa-static

使用同express中的static类似

 2const Koa = require('koa')
 3const static = require('koa-static')
 4
 5const app = new Koa()
 6
 7// app.use(static(__dirname + '/public'))

错误处理

koa中错误处理与express中的错误处理有些不同，koa中next()方法不能传递错误信息，只能借助于ctx.app.emit来传递错误信息。使用app.on来监听对应的错误事件信息。

 1const path = require('path')
 2const Koa = require('koa')
 3const Router = require('@koa/router')
 4const bodyParser = require('koa-bodyparser')
 5
 6const app = new Koa()
 7const router = new Router()
 8
 9app.use(bodyParser())
10
11router.post('/login', (ctx, next) => {
12	const { username, password } = ctx.request.body
13	if (!username || !password) {
14		ctx.app.emit('error', -1001, ctx)
15	} else if (username !== 'admin' || password !== '123123') {
16		ctx.app.emit('error', -1002, ctx)
17	} else {
18		ctx.body = {
19			code: 0,
20			msg: `登录成功！欢迎回来，${username}`
21		}
22	}
23})
24
25app.on('error', (code, ctx) => {
26	let msg = ''
27
28	switch (code) {
29		case -1001:
30			msg = '用户名或密码不能为空'
31			break
32		case -1002:
33			msg = '用户名或密码错误'
34			break
35		default:
36			msg = '未知错误'
37	}
38
39	ctx.body = {
40		code,
41		msg
42	}
43})
44
45app.use(router.routes())
46
47app.listen(3000, () => {
48	console.log('server is running at http://localhost:3000')
49})

Koa与Express中间件上的区别

koa的中间件在同步于异步上都支持洋葱模型，在request时会按中间件顺序传递数据，并在response时按照中间件的逆序返回对应的数据。

express仅在同步上支持洋葱模型，异步并不支持该方式。

koa同步模式下传递数据

 1const Koa = require('koa')
 2
 3const app = new Koa()
 4
 5// koa同步模式下中间件传递数据
 6app.use((ctx, next) => {
 7	console.log('middleware 1 start')
 8	ctx.msg = 'md 1 + '
 9	next()
10	ctx.body = ctx.msg
11	console.log('middleware 1 end')
12})
13
14app.use((ctx, next) => {
15	console.log('middleware 2 start')
16	ctx.msg += 'md 2 + '
17	next()
18	console.log('middleware 2 end')
19})
20
21app.use((ctx, next) => {
22	console.log('middleware 3 start')
23	ctx.msg += 'md 3'
24	console.log('middleware 3 end')
25})
26
27app.listen(3000, () => {
28	console.log('server is running at http://localhost:3000')
29})
30
31// 输出结果：
32// middleware 1 start
33// middleware 2 start
34// middleware 3 start
35// middleware 3 end
36// middleware 2 end
37// middleware 1 end
38
39// 客户端输出：
40// md 1 + md 2 + md 3

异步模式传递数据

 1const Koa = require('koa')
 2const axios = require('axios')
 3
 4const app = new Koa()
 5
 6// koa异步模式下中间件传递数据
 7app.use(async (ctx, next) => {
 8	console.log('middleware 1 start')
 9	ctx.msg = 'md 1 + '
10	await next()
11	ctx.body = ctx.msg
12	console.log('middleware 1 end')
13})
14
15app.use(async (ctx, next) => {
16	console.log('middleware 2 start')
17	ctx.msg += 'md 2 + '
18	await next()
19	console.log('middleware 2 end')
20})
21
22app.use(async (ctx, next) => {
23	console.log('middleware 3 start')
24	const res = await axios.get('https://jsonplaceholder.typicode.com/users/1')
25
26	ctx.msg += res.data.name
27	console.log('middleware 3 end')
28})
29
30app.listen(3000, () => {
31	console.log('server is running at http://localhost:3000')
32})
33
34// 输出结果：
35// middleware 1 start
36// middleware 2 start
37// middleware 3 start
38// middleware 3 end
39// middleware 2 end
40// middleware 1 end
41
42// 客户端输出：
43// md 1 + md 2 + Leanne Graham

可以看到koa在同步和异步模式下中间件均支持洋葱模型输出数据。

express仅在同步模式下同koa一样，异步模式下next()并不支持promise的方式，会直接跳过还在await的数据，进入到下一个中间件。

所以在使用express异步只能在最后的中间件中获取到最终的数据，不支持在所有中间中都能获取到最终数据。
