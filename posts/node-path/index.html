Node 01 Path 模块

Simz
2023-10-08
node

path（路径）模块，用于处理文件和目录路径。

path.normalize(path)

  • path <string>
  • 返回： <string>

path.normalize() 方法规范化给定的 path，解析..和 .片段。

示例：

1const path = require('path')
2
3const filepath = path.normalize('///a//b//c//d//e//..//f//g//')
4
5console.log('filepath', filepath) //filepath \a\b\c\d\f\g\

path.join([…paths])

  • ...paths <string> 路径片段的序列
  • 返回： <string>

path.join() 方法使用特定于平台的分隔符作为定界符将所有给定的 path 片段连接在一起，然后规范化生成的路径。零长度的 path 片段被忽略。 如果连接的路径字符串是零长度字符串，则将返回 .，表示当前工作目录。

示例：

1const path = require('path')
2
3console.log('path:', path.join('/foo', 'bar', 'baz/asdf', 'quux', '..'))
4// path: \foo\bar\baz\asdf
5
6// 如果路径中有./或者../会进行忽略
7console.log('path:', path.join('foo', '/bar', './baz/asdf', '../quux'))
8// path: foo\bar\baz\quux

path.resolve([…paths])

  • ...paths <string> 路径片段的序列
  • 返回： <string>

path.resolve() 方法将路径或路径片段的序列解析为绝对路径。

给定的路径序列从右到左处理，每个后续的 path 会被追加到前面，直到构建绝对路径。

需要注意一下三种情况

  1. 如果有/开头的路径，则会使用最后一个/开头的路径作为根路径进行输出，并忽略前面的路径
  2. 如果路径中有./或者../将会忽略，直接进行拼接
  3. 如果路径中没有/路径，则会返回一个当前文件所在目录与路径的拼接。
1const path = require('path')
2
3console.log('path:', path.resolve('/foo/bar', './baz')) // path: o:\foo\bar\baz
4console.log('path:', path.resolve('foo/bar', '/baz')) //path: o:\baz
5console.log('path:', path.resolve('foo/bar', './baz')) //path: o:\NodeJs\node-demo\foo\bar\baz

path.isAbsolute(path)

  • path <string>
  • 返回： <boolean>

path.isAbsolute() 方法判断 path 是否为绝对路径。

如果给定的 path 是零长度字符串，则将返回 false

示例：

1const path = require('path')
2
3console.log(path.isAbsolute('/foo/bar')) // true
4console.log(path.isAbsolute('/baz/..')) // true
5console.log(path.isAbsolute('o:/foo')) // true
6console.log(path.isAbsolute('o://foo//bar')) // true
7console.log(path.isAbsolute('qux/')) // false
8console.log(path.isAbsolute('qux\\baz')) // false
9console.log(path.isAbsolute('.')) // false

path.basename(path[, suffix])

  • path <string>
  • suffix <string> 要删除的可选后缀
  • 返回： <string>

path.basename() 方法返回 path 的最后一部分，类似于 Unix basename 命令。

示例：

1const path = require('path')
2
3console.log(path.basename('/foo/bar/baz/asdf/quux.html')) // quux.html
4console.log(path.basename('/foo/bar/baz/asdf/quux.html', '.html')) // quux
5console.log(path.basename('/foo/bar/baz/asdf/quux.HTML', '.html')) // quux.HTML

path.extname(path)

  • path <string>
  • 返回： <string>

path.extname() 方法返回 path 的扩展名，即 path 的最后一部分中从最后一次出现的 .（句点）字符到字符串的结尾。

如果 path 的最后一部分中没有 .，或者除了 path 的基本名称（参见 path.basename()）的第一个字符之外没有 . 个字符，则返回空字符串。

示例：

1const path = require('path')
2
3console.log(path.extname('demo03.js')) // .js
4console.log(path.extname('demo03.js.txt')) // .txt
5console.log(path.extname('demo03')) // ''
6console.log(path.extname('demo03.')) // '.'
7console.log(path.extname('demo03.js/')) // '.js'
8console.log(path.extname('.js')) // ''
9console.log(path.extname('.js.ts')) // .ts

path.dirname(path)

  • path <string>
  • 返回： <string>

path.dirname() 方法返回 path 的目录名，类似于 Unix dirname 命令。 尾随的目录分隔符被忽略。

示例：

1const path = require('path')
2
3console.log(path.dirname('/foo/bar/baz/asdf/quux')) // /foo/bar/baz/asdf
4console.log(path.dirname('/foo/bar/baz/asdf/')) // /foo/bar/baz
5console.log(path.dirname('/foo/bar/baz/asdf/quux.html')) // /foo/bar/baz/asdf

path.parse(path)

  • path <string>
  • 返回： <string>

path.parse() 方法返回一个对象，其属性表示 path 的重要元素。 尾随的目录分隔符被忽略。

返回的对象将具有以下属性：

  • dir
  • root
  • base
  • name
  • ext

示例：

 1const path = require('path')
 2
 3console.log(path.parse('/foo/bar/baz/asdf/quux.txt'))
 4// {
 5//   root: '/',
 6//   dir: '/foo/bar/baz/asdf',
 7//   base: 'quux.txt',
 8//   ext: '.txt',
 9//   name: 'quux'
10// }

path.format(pathObject)

  • pathObject <object>
    • dir
    • root
    • base
    • name
    • ext
  • 返回 <string>

示例：

 1const path = require('path')
 2
 3const pathObj = {
 4	root: '/',
 5	dir: '/foo/bar/baz/asdf',
 6	base: 'quux.txt',
 7	ext: '.txt',
 8	name: 'quux'
 9}
10console.log(path.format(pathObj)) // /foo/bar/baz/asdf/quux.txt
