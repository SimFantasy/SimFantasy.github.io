ElementPlus的坑

Simz
2022-05-05
vue ElementPlus 避坑指南

记录一下在使用ElementPlus中的一些坑。

按需导入

坑：

  1. 之前没有这两个插件，不能自动导入，需要通过配安装babel，通过babel.config.js配置，组件需要统一全局手动注册一次，并且需要引入element-plus的基础样式。
  2. 按官方说明安装后，引入组件无效，卸载之前的element-plus和unplugin-vue-components、unplugin-auto-import。
  3. 重新进行安装，将配置放到webpack中。vue.config.ts中放入后会报错，不存在plugins的方法。
  4. 通过插件自动导入组件无需引入基础样式。

自动导入

需要安装unplugin-vue-components 和 unplugin-auto-import这两款插件

1npm install -D unplugin-vue-components unplugin-auto-import

然后在vite或者webpack中进行配置。

Vite
 1// vite.config.ts
 2import AutoImport from 'unplugin-auto-import/vite'
 3import Components from 'unplugin-vue-components/vite'
 4import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'
 5
 6export default {
 7  plugins: [
 8    // ...
 9    AutoImport({
10      resolvers: [ElementPlusResolver()],
11    }),
12    Components({
13      resolvers: [ElementPlusResolver()],
14    }),
15  ],
16}
webpack
 1// webpack.config.js
 2const AutoImport = require('unplugin-auto-import/webpack')
 3const Components = require('unplugin-vue-components/webpack')
 4const { ElementPlusResolver } = require('unplugin-vue-components/resolvers')
 5
 6module.exports = {
 7  // ...
 8  plugins: [
 9    AutoImport({
10      resolvers: [ElementPlusResolver()],
11    }),
12    Components({
13      resolvers: [ElementPlusResolver()],
14    }),
15  ],
16}

全局引入的样式

坑：

  1. 全局引入样式，原来路径为import 'element-plus/lib/theme-chalk/index.css' ，基础样式引入为import 'element-plus/lib/theme-chalk/base.css'
  2. 2.0以后官方修改为 import 'element-plus/dist/index.css'

Loading组件

坑：

  1. loading组件在进行axios封装时使用，给所有axios封装的实例添加请求拦截器成功时，加载loading组件，在响应成功后关闭该组件。
  2. 在封装中需要配置 loading属性的类型，原来为 import { ILoadingInstance } from 'element-plus/lib/el-loading/src/loading.type'
  3. 现在属性类型和位置为import { LoadingInstance } from 'element-plus/lib/components/loading/src/loading'
