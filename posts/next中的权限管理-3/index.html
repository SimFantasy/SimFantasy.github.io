NextAuth配置

next-auth的配置很麻烦，需要根据步骤一步步来。

env

首先，也是使用auth.js的命令生成NEXTAUTH_SECRET，添加到.env中。

之前还需要配置NEXTAUTH_JWT_SECRET但这个在配置中已经弃用了。NEXTAUTH_URL这个有时会莫名报错，需要你补充这个。

1# Next Auth
2NEXTAUTH_SECRET=
3# NEXTAUTH_JWT_SECRET=
4# NEXTAUTH_URL=
Next Options

在配置完.env之后我们来配置next-auth

 1// src/lib/auth/auth-options.ts
 2
 3import { AuthOptions } from 'next-auth'
 4import Credentials from 'next-auth/providers/credentials'
 5
 6import { db } from '@/lib/db'
 7import { loginSchema } from '@/lib/schemas'
 8import { comparePassword } from '@/lib/utils'
 9
10export const authOptions: AuthOptions = {
11	providers: [
12		Credentials({
13			name: 'Credentials',
14			credentials: {
15				email: { label: 'Email', type: 'text' },
16				password: { label: 'Password', type: 'password' }
17			},
18			async authorize(credentials) {
19				const validatedValues = loginSchema.safeParse(credentials)
20				if (!validatedValues.success) return null
21				const { email, password } = validatedValues.data
22
23				try {
24					const existingUser = await db.user.findUnique({ where: { email } })
25					if (!existingUser || !existingUser.password) return null
26
27					const isMatch = comparePassword(password, existingUser.password)
28					if (!isMatch) return null
29
30					return { id: existingUser.id, name: existingUser.name, email: existingUser.email }
31				} catch (error) {
32					console.log('Error while logging in', error)
33					return null
34				}
35			}
36		})
37	],
38	pages: {
39		signIn: '/login'
40	},
41	session: { strategy: 'jwt' },
42	secret: process.env.NEXTAUTH_SECRET!,
43	callbacks: {
44		async jwt({ token, user }: { token: any; user: any }) {
45            // if(token){
46			// 	token.id = user.id
47			// 	token.name = user.name
48			// 	token.email = user.email
49			// 	token.image = user.image
50			// }
51            
52			if (!token.sub) return token
53
54			const existingUser = await db.user.findUnique({ where: { id: token.sub } })
55			if (!existingUser) return token
56
57			token.id = existingUser.id
58			token.name = existingUser.name
59			token.email = existingUser.email
60			token.image = existingUser.image
61
62			return token
63		},
64		async session({ session, token }: { session: any; token: any }) {
65			if (token && session.user) {
66				session.user.id = token.id
67				session.user.name = token.name
68				session.user.email = token.email
69				session.user.image = token.image
70			}
71
72			return session
73		}
74	}
75}

providers可以放Credentials或者Github、Google的Provider，本篇文章中暂不涉及第三方登录。

这里需要注意authorize里的内容是用来进行登录的action，但返回值不能为返回的错误对象信息，如之前我们定义的{error: boolean, message: string}，仅能返回Awaitable<User | null>需要注意下。

authorize里返回的user对象，会传递给callbacks中的jwt方法中的user参数，jwt方法中的token参数中的sub会得到传递过来的user.id，所以这里既可以使用user参数直接传递user信息给token，也可以直接在数据库里查询后，将数据库中的信息传递给token。

接下来有需要将token信息传递给session，这里的写法基本就是一个模板代码，按照此结构可以增加保存到session中的字段。

这里在多说一嘴，官方的说明里authOptions是放到app/api/auth/[...nextauth]/route.ts这路由文件中的，因为很多情况下需要引用这个配置，就需要引用这个路由，比如：getServerSession，非常的别扭。所以很多情况下都是把这个authOptions单独提取出来的。这里可以查看官方的说明： Getting Started

Typescript

如果使用typescript时，还需要注意，因为next-auth对session的字段定义只有name、emial、image，连id都没有，所以在使用session.user相关的类型时，会报类型错误。所以需要来扩展下类型，具体可以查看官方说明：TypeScript

 1// src/lib/auth/auth.d.ts
 2import { DefaultSession } from 'next-auth'
 3
 4declare module 'next-auth' {
 5	interface Session {
 6		user: {
 7			id: string
 8		} & DefaultSession['user']
 9	}
10}

注意：这里需要联合DefaultSession['user']这个类型，否则在使用时，会找不到对应的name、email、image的类型定义

至此，关于next-auth的配置基本完成了。

接下来完成注册、登录，首先来写actions的部分，因为登录部分已经集成到authOptions的配置中了，所以此处只需要写注册的action。

另外，这里还写了服务端获取currentUser和通过id获取用户，后续都会用到。

 1// src/actions/user.ts
 2
 3'use server'
 4
 5import { z } from 'zod'
 6import { db } from '@/lib/db'
 7import { getServerSession } from 'next-auth'
 8
 9import { authOptions } from '@/lib/auth/auth-options'
10import { registerSchema } from '@/lib/schemas'
11import { IResponse, ISessionUser } from '@/lib/types'
12import { ActionResponse } from '@/lib/action-response'
13import { hashPassword } from '@/lib/utils'
14
15export const getCurrentUser = async (): Promise<IResponse<ISessionUser>> => {
16	const session = await getServerSession(authOptions)
17
18	if (!session?.user?.email) {
19		return ActionResponse.badRequest('User not authenticated.')
20	}
21
22	const existingUser = await db.user.findUnique({
23		where: { email: session.user.email }
24	})
25	if (!existingUser) {
26		return ActionResponse.notFound('User not found.')
27	}
28
29	return ActionResponse.ok({
30		data: {
31			id: existingUser.id,
32			name: existingUser.name,
33			email: existingUser.email,
34			image: existingUser.image || '/images/default-avatar.png'
35		},
36		message: 'Get user successful.'
37	})
38}
39
40export const getUserById = async (id: string) => {
41	return await db.user.findUnique({
42		where: { id }
43	})
44}
45
46export const registerAction = async (
47	values: z.infer<typeof registerSchema>
48): Promise<IResponse<ISessionUser>> => {
49	const validatedValues = registerSchema.safeParse(values)
50	if (!validatedValues.success) {
51		return ActionResponse.badRequest('Invalid input data.')
52	}
53
54	const { username, email, password } = validatedValues.data
55
56	try {
57		const existingUser = await db.user.findUnique({
58			where: { email }
59		})
60		if (existingUser) {
61			return ActionResponse.badRequest('Email already registered.')
62		}
63
64		const hashedPassword = hashPassword(password)
65
66		const user = await db.user.create({
67			data: {
68				name: username,
69				email,
70				password: hashedPassword
71			}
72		})
73
74		return ActionResponse.ok({
75			data: {
76				id: user.id,
77				name: user.name,
78				email: user.email,
79				image: user.image || '/images/default-avatar.png'
80			},
81			message: 'Registration successful.'
82		})
83	} catch (error) {
84		console.log('Register error:', error)
85		return ActionResponse.badRequest('Registration failed.')
86	}
87}

Middleware

登录注册完成后，我们还需要完善下middleware.ts，这个和之前的差不多，仅需调整下isLoggedIn的定义。

 1// src/middleware.ts
 2import { NextRequest, NextResponse } from 'next/server'
 3import { getToken } from 'next-auth/jwt'
 4
 5import {
 6	publicRoutes,
 7	authRoutes,
 8	apiAuthPrefix,
 9	DEFAULT_LOGIN_REDIRECT
10} from '@/lib/auth/auth-routes'
11
12export async function middleware(req: NextRequest) {
13	const { nextUrl } = req
14	const token = await getToken({ req })
15	const isLoggedIn = !!token
16
17	const isApiAuthRoute = nextUrl.pathname.startsWith(apiAuthPrefix)
18	const isPublicRoute = publicRoutes.includes(nextUrl.pathname)
19	const isAuthRoute = authRoutes.includes(nextUrl.pathname)
20
21	if (isApiAuthRoute) return null
22
23	if (isAuthRoute) {
24		if (isLoggedIn) {
25			return NextResponse.redirect(new URL(DEFAULT_LOGIN_REDIRECT, nextUrl))
26		}
27		return null
28	}
29
30	if (!isLoggedIn && !isPublicRoute) {
31		return NextResponse.redirect(new URL('/login', nextUrl))
32	}
33
34	return null
35}
36
37export const config = {
38	matcher: ['/((?!.*\\..*|_next).*)', '/', '/(api|trpc)(.*)']
39}

NextAuth在服务端和客户端的方法

客户端方法：

useSession()

NextAuth.js客户端中的useSession() React Hook是检查某人是否登录的最简单方法。

 1import { useSession } from "next-auth/react"
 2
 3export default function Component() {
 4  const { data: session, status } = useSession()
 5
 6  if (status === "authenticated") {
 7    return <p>Signed in as {session.user.email}</p>
 8  }
 9
10  return <a href="/api/auth/signin">Sign in</a>
11}
  • data：可以有三个值：Session | null | undefined
  • status：enum有三种会话状态：loading | authenticated | unauthenticated

getSession()

NextAuth.js提供了一个getSession()，应该只在客户端调用它来返回当前的活动会话。

1async function myFunction() {
2  const session = await getSession()
3  /* ... */
4}

这个通常用的比较少，官方说可以在服务端使用，但在服务端使用getSession()概率获取不到session只能得到null。

总结

优点：

  • 功能无疑是之前两个项目不能比的，从配置到UI都可以自定义
  • 在next.js 15的版本中可用

缺点：

  • next-auth配置相对比较麻烦，要配置authOptions，配置路由app/api/auth/[...nextaut]/route.ts，配置AuthProvider、配置middleware.ts等
  • 文档稀烂，阅读非常痛苦
  • 同样的配置，在不同项目里会莫名抽风报错。

总结：

next-auth的v4版本，相对来说比较稳定适合比较大的项目，但是配置很麻烦。
