Node 05 Stream 流

Simz
2023-10-08
Node

程序中的stream(流)可以想象当我们从一个文件中读取数据时，文件的二进制（字节）数据会源源不断的被读取到我们程序中，而这个一连串的字节，就是我们程序中的流。

  • 是连续字节的一种表现形式和抽象概念
  • 流应该是可读的，也是可写的

为什么需要流？

直接通过 readFile或者 writeFile方式读写文件的问题。

  • 直接读写文件的方式，虽然简单，但是无法控制一些细节的操作，比如从什么位置开始读、读到什么位置、一次性读取多少个字节。
  • 读到某个位置后，暂停读取，某个时刻恢复读取。
  • 或者这个文件非常大，比如一个视频文件，一次性全部读取并不合适。

Node中很多对象是基于流实现的，http模块的Request和Response对象，process.stdout对象。所有的流都是EventEmitter的实例。

Node.js中有四种基本流类型

  • Writable：可以向其写入数据的流（例如 fs.createWriteStream()）。
  • Readable：可以从中读取数据的流（例如 fs.createReadStream()）。
  • Duplex：同时为Readable和的流Writable（例如 net.Socket）。
  • Transform：Duplex可以在写入和读取数据时修改或转换数据的流（例如zlib.createDeflate()）。

fs.createReadStream(path[, options])

  • path <string> | <Buffer> | <URL>
  • options <string> | <Object>
    • flags <string> 参见 支持文件系统 flags。 默认值： 'r'
    • encoding <string> ，文件编码，默认值：null
    • fd <integer> | <FileHandle> 默认值： null
    • mode <integer> 默认值： 0o666
    • autoClose <boolean> 默认值：true
    • emitClose <boolean> 默认值： true
    • start <integer>，文件读取开始位置
    • end <integer> ， 文件读取结束位置，默认值： Infinity
    • highWaterMark <integer> 一次性读取字节的长度，默认值： 64 * 1024
    • fs <Object> | <null> 默认值： null
  • 返回： <fs.ReadStream>

如果指定了 fd，则 ReadStream 将忽略 path 参数并使用指定的文件描述符。 这意味着不会触发 'open' 事件。

如果 fd 指向仅支持阻塞读取的字符设备（例如键盘或声卡），则读取操作不会在数据可用之前完成。 这可以防止进程退出和流自然关闭。

默认情况下，流将在销毁后触发 'close' 事件。 将 emitClose 选项设置为 false 以更改此行为。

示例：

通过fs中的createReadStream，读取文件中的内容

 1const fs = require('fs')
 2
 3// 创建可读流
 4const readstream = fs.createReadStream('./demo.txt')
 5
 6// 获取文件数据
 7readstream.on('data', data => {
 8	console.log('data:', data) // data: <Buffer 68 65 6c 6c 6f 20 6e 6f 64 65 0a>
 9	console.log('data:', data.toString()) // data: hello node
10})

通过配置start、end、highWaterMark来控制读取文件中的内容。

注意：如果未配置encoding为utf8则在data中使用toString()方法会报错，createReadStream默认编码为null，toString()中默认编码为utf8

 1const fs = require('fs')
 2
 3// 创建可读流
 4const readstream = fs.createReadStream('./demo.txt', {
 5	encoding: 'utf8',
 6	start: 3,
 7	end: 8,
 8	highWaterMark: 2
 9})
10
11// 通过监听data事件，获取读取到的数据
12readstream.on('data', data => {
13	console.log('data:', data.toString())
14})
15
16// data: lo
17// data: n
18// data: od

fs.createWriteStream(path[, options])

  • path <string> | <Buffer> | <URL>
  • options <string> | <Object>
    • flags <string> 参见 支持文件系统 flags。 默认值： 'w'
    • encoding <string> 默认值： 'utf8'
    • fd <integer> | <FileHandle> 默认值： null
    • mode<integer> 默认值： 0o666
    • autoClose <boolean> 默认值： true
    • emitClose <boolean> 默认值： true
    • start <integer>
    • fs <Object> | <null> 默认值：null
  • 返回： <fs.WriteStream>

start 选项，以允许在文件开头的某个位置写入数据，允许的值在 [0, Number.MAX_SAFE_INTEGER] 范围内。

修改文件而不是替换它可能需要将 flags 选项设置为 r+ 而不是默认的 w。

如果将 autoClose 设置为 true（默认行为），则在 'error' 或 'finish' 时文件描述符将自动关闭。

如果 autoClose 为 false，则即使出现错误，文件描述符也不会关闭。 关闭它并确保没有文件描述符泄漏是应用的责任。

默认情况下，流将在销毁后触发 'close' 事件。 将 emitClose 选项设置为 false 以更改此行为。

如果 options 是字符串，则它指定编码。

示例：

注意：使用flags: 'a+'，start: 3，按开始位置写入，在macOS环境中执行没有问题，但在windows环境中需要使用r+来进行位置写入，否则只会在最后进行内容追加。

 1const fs = require('fs')
 2
 3// 写入文件
 4const writeStream = fs.createWriteStream('./demo2.txt', {
 5	flags: 'a+',
 6	start: 3
 7})
 8
 9// 监听写入完成
10writeStream.write('hello world\n', err => {
11	if (err) {
12		console.log(err)
13	} else {
14		console.log('写入成功')
15	}
16})

通过write方法多次写入，通过end方法完成最后的写入，并通过finish事件监听写入文件结束，close事件监听写入文件关闭。

通过Pipe传递流

通过createReadStream来读取文件，通过pipe (管道)来传递读取的buffer内容，传递给createWriteStream写入文件。

示例：

 1const fs = require('fs')
 2
 3// 1. 读取文件
 4const readStream = fs.createReadStream('./demo2.txt')
 5
 6// 2. 写入文件
 7const writeStream = fs.createWriteStream('./demo.txt', {
 8	flags: 'a+'
 9})
10
11// 3. 管道流
12readStream.pipe(writeStream)
13
14// 4. 监听读取流
15readStream.on('end', err => {
16	if (err) {
17		console.log('读取失败')
18	} else {
19		console.log('读取成功')
20	}
21})
22
23// 5. 监听写入流
24writeStream.on('finish', err => {
25	if (err) {
26		console.log('写入失败')
27	} else {
28		console.log('写入成功')
29	}
30})
31
32// 读取成功
33// 写入成功
