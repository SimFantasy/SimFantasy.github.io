Strapi用户登录鉴权

Simz
2022-10-12
React Strapi

延续之前Strapi的使用，Strapi已经做了很完整的权限，并且给了API，所以结合之前的博客内容，完成对应的登录、注册与鉴权。

之前博客在公共模式Public下是可以直接访问到对应的文章，这次我们取消Public的find和findone的权限。并给Authenticated配置对应的Post权限。在Strapi中调整完成后开始代码。

‍

技术栈：

Valtio作为状态库。一个简单好用的玩意。

Axios作为请求工具。后续会对其进行封装。

React-query结合axios作为数据请求库。

Vite作为构建工具，并使用vite-pretty-lint快速配置ESLint和Prettier。

Styled-components作为Css In JS的样式工具。

Styled-normalize做样式初始化。

React-router-dom是路由工具。

Dayjs作为日期格式化工具。

Polished是一个对Styled-components样式加强的工具。从一个外国大佬的视频里看到的，Cool~本文并未提到。

‍

项目很小，用到的东西也不多。

CheckAuth

 1import React from 'react'

 4import { authState } from '@/store'
 5
 6const CheckAuth = props => {
 7  const snap = useSnapshot(authState)
 8  const pathname = useLocation().pathname
 9  const isToken = localStorage.getItem('token')
10
11  return snap.isLogin && isToken ? ( 
12    props.children
13  ) : (
14    <Navigate to='/login' state={{ preLocation: pathname }} replace />
15  )
16}
17
18export default CheckAuth

这里使用isLogin是否为true和isToken是否获取到了localstorage中的token作为判断，如果都为true则可以访问后面的children子组件。如果没有访问到则会跳转到登录。

另外这里做了一个路径记录，Navigate中的state将当前路径记录，并传递给下个路由。如果登录成功，则会跳回到之前的路径。这里有点拗口，简单说就是，如果是从首页登录的登录后跳回首页。如果从用户信息页面访问因为没有权限调到了登录，登录后又会跳回到用户信息。

fetcher

这个是用axios进行封装的请求方法。是这次我学习的主要关键点部分

  1. 通过请求拦截，给headers中的Authorization注入token，进行标识。
  2. 通过响应拦截，获取服务器传递过来的错误，并抛出，提供给react-query。
  3. 这里在查找资料时还看到应该有在响应代码为403时，要重新刷新token这次并没有做。
 1// src/service/fetcher.js
 2
 3import axios from 'axios'
 4
 5const createInstence = () => {
 6  const instence = axios.create({
 7    baseURL: 'http://localhost:1337/api',
 8    timeout: 50000
 9  })
10  instence.interceptors.request.use(handleToken)
11  instence.interceptors.response.use(handleResponse, handleError)
12
13  return instence
14}
15
16const handleToken = config => {
17  const token = localStorage.getItem('token')
18  if (token) {
19    config.headers.Authorization = `Bearer ${token}`
20  }
21  return config
22}
23
24const handleResponse = response => {
25  return response.data
26}
27
28const handleError = error => {
29  const { message, response } = error
30  return Promise.reject(response ? response.data.error.message || message : error)
31}
32
33const fetcher = createInstence()
34
35export default fetcher

注意handleError中的response.data.error.message结构，这个是strapi返回的错误消息。

store

这个是本次的关键点。可以看到对比redux和mobx一对的模板代码这个valtio简洁太多。

  1. 先获取下token，看token是否存在，通过token来进行判断初始值。

  2. 因为strapi传过来的user信息是一个对象，所以需要进行序列化转成json放到localstorage。

  3. 然后封装两个方法。

    1. login时，通过state传入登录后获取到的token和user信息，并将对应的信息存入localstorage。
    2. 记录登录的时间currentTime，与设置的到期时间进行相加得到token过期时间expirationTime保存到localstorage。在app.js中有一个useAutoLogout的hooks，通过这个来进行判断是否到期登出。最后再来说这个钩子函数。
    3. logout时，清空store的初始值，清空localstorage中的内容。
 1// src/store/index.js
 2
 3import { proxy } from 'valtio'
 4
 5const token = localStorage.getItem('token')
 6
 7export const authState = proxy({
 8  isLogin: token ? true : false,
 9  token: token ? token : null,
10  user: token ? JSON.parse(localStorage.getItem('user')) : {},
11  expirationTime: token ? Number(localStorage.getItem('expirationTime')) : 0 // 登录状态失效时间
12})
13
14export const authAction = {
15  login: state => {
16    authState.isLogin = true
17    authState.token = state.token
18    authState.user = state.user
19
20    const currentTime = Date.now() // 当前时间
21    const timetout = 1000 * 60 * 60 * 24 * 7 // 有效时间 一周
22    authState.expirationTime = currentTime + timetout //设置失效日期 一周后失效
23
24    localStorage.setItem('token', state.token)
25    localStorage.setItem('user', JSON.stringify(state.user))
26    localStorage.setItem('expirationTime', Number(authState.expirationTime))
27  },
28  logout: state => {
29    authState.isLogin = false
30    authState.token = null
31    authState.user = []
32    authState.expirationTime = 0
33
34    localStorage.removeItem('token')
35    localStorage.removeItem('user')
36    localStorage.removeItem('expirationTime')
37  }
38}

register

注册，这个相对于登录比较简单，只是将用户信息提交给strapi，如果有错误strapi就会返回过来。

  1. 通过ref获取到表单信息，这里不是受控组件了。受控组件比较麻烦，需要做防抖。否则就需要装一些包来处理抖动的问题。
  2. 通过react-query来处理表单提交的状态，并在成功后返回提示，设置登出后跳转路由。如果注册错误则返回错误信息。
  3. 通过绑定在表单上的onSubmit的handleSubmit来提交表单信息，将获取到的表单内容传递给react-query的useMutation中的一个方法，这个方法将用户信息传递给api。api里会结构用户信息传递给strapi。
  4. 由于没有用UI库，仅对返回的错误和成功的消息做了简单的样式区分和固定显示位置。

useAutoLogout

最后来看下自动到期的useAutoLogout，这个本身不算是一个真正的钩子，只是一个到期函数。

  1. 获取到store中的 expirationTIme，减去 当前时间，得到剩余有效时间，通过这个与一个阈值(1分钟 ：6000ms)进行判断。
  2. 如果小于1分钟则直接登出，如果到期登出。
 1import React, { useEffect } from 'react'
 2import { useSnapshot } from 'valtio'
 3import { authState, authAction } from '@/store'
 4
 5const useAutoLogout = () => {
 6  const snap = useSnapshot(authState)
 7
 8  useEffect(() => {
 9    const timeout = snap.expirationTime - Date.now() //到期时间 - 当前时间 = 剩余有效时间
10
11    if (timeout < 6000) {
12      authAction.logout() //如果剩余有效时间小于1分钟，直接登出
13      return
14    }
15
16    const timer = setTimeout(() => {
17      authAction.logout() // 到期时间后，登出
18    }, timeout)
19
20    return () => {
21      clearTimeout(timer)
22    }
23  }, [snap.expirationTime])
24}
25
26export default useAutoLogout
