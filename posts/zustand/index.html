React Zustand

Zustand 是一个轻量级、简洁且强大的 React 状态管理库，旨在为 React 项目提供更简单、更灵活的状态管理方式。与其他流行的状态管理库（如 Redux、MobX 等）相比，Zustand 的 API 更加简洁明了，学习成本较低，且无需引入繁琐的中间件和配置。

安装

1pnpm add zustand

基础使用

 1// src/store/baseStore.ts
 2import { create } from 'zustand'
 3
 4type BearState = {
 5  bears: number
 6  increasePopulation: () => void
 7  removeAllBears: () => void
 8}
 9
10const useBearStore = create<BearState>()(set => ({
11  bears: 0,
12  increasePopulation: () =>
13    set(state => ({
14      bears: state.bears + 1
15    })),
16  removeAllBears: () => set({ bears: 0 })
17}))
18
19export default useBearStore
  • 在ts模式下传类型给create方法时，需要注意多出来的括号写法create<BearState>()({...})。
  • 在zustand中大部分情况下都是使用的回调函数获取数据，所以需要注意
    • 返回值的括号嵌套写法，()=>({})返回的是一个对象，()=>{}后面的大括号是一个表达式，没有明确返回值，除非在里面写上return。
    • 因为使用了很多回调函数的方式，所以函数的嵌套会很多，可以使用ctrl+shift+->+->...的方式选中同一个函数表达式。
  • 在set(stete=>({ bears: state.bears + 1}))中，正常写法应该为set(state => ({ ...state, bears: state.bears + 1}))需要先将state结构，再替换其中的bears。这里是因为zustand在第一层自动进行了合并，但如果存在多层嵌套，则需要使用展开方式来书写。也可以后面使用immer来实现更简便的写法。

17          Add Bear
18        </button>
19        <button className='btn' onClick={removeAllBears}>
20          Remove All Bears
21        </button>
22      </div>
23    </div>
24  )

26
27export default BearBox

上述代码中，通过useBearStore传入回调的方式获取到对应的状态和处理函数。

  • 如果有更多的内容需要从store中取出，非常的麻烦。所以zustand还可以写成 const { bears, increasePopulation, removeAllBears} = useBearStroe()的方式将store中所有内容一次性取出来。但这样又会造成不必要的重渲染。（后面会来进行说明重渲染问题）

  • 也可以使用自动获取选择器和多个状态选择器的方式取出，后面再做说明。

Set、immer的使用

使用set时需要将set中的state当做不可变状态使用，zustand会自动合并第一层状态，但状态嵌套层次较多时，就需要使用解构后替换对应状态的方式来操作。

示例：

将cat区分为bigCat和smallCat来进行计数。

 1// src/store/cateState-summary.ts
 2import { create } from 'zustand'
 3
 4type CatState = {
 5  cats: {
 6    bigCats: number
 7    smallCats: number
 8  }
 9  increaseBigCat: () => void
10  increaseSmallCat: () => void
11}
12
13const useCatStore = create<CatState>()({
14  set => ({
15  	cats: {
16  		bigCats: 0,
17  		smallCats: 0,
18		},
19    increaseBigCat: () => set(state => ({ ...state, cats: { ...state.cats, bigCats: state.cats.bigCats + 1}})),
20    increaseSmallCat: () => set(state => ({ ...state, cats: { ...state.cats, smallCats: state.cats.smallCats + 1}}))
21	})
22})

上述代码，因为cats有多个层级，所以在写actions时，解构让代码变的很复杂。这时可以引入immer这个库来处理，Immer 中间件使您能够以更方便的方式使用不可变状态。此外，借助 Immer，可以简化在 Zustand 中处理不可变数据结构的过程。

immer中间件

安装
1pnpm add immer
使用
 1// src/store/cateState-summary.ts
 2import { create } from 'zustand'
 3import { immer } from 'zustand/middleware/immer'
 4
 5type CatState = {
 6  cats: {
 7    bigCats: number
 8    smallCats: number
 9  }
10  increaseBigCat: () => void
11  increaseSmallCat: () => void
12}
13
14const useCatStore = create<CatState>()({
15  set => ({
16  	cats: {
17  		bigCats: 0,
18  		smallCats: 0,
19		},
20    increaseBigCat: () => set(state => { state.cats.bigCats++ }),
21    increaseSmallCat: () => set(state => { state.cats.smallCats++ })
22	})
23})

上述代码中，在安装了immer，需要注意从zustand/middleware/immer中进行引用。在使用时可以直接对属性进行计数而无需多次解构。

注意：immer会默认将state进行返回，所以写法上还需要写在{}中，不能省略。

get的使用

如果现在set之外使用state时，就需要用到get来获取state，而不能直接使用state。

示例：

增加一个cats的总计。

 1// src/store/catState-get.ts
 2import { create } from 'zustand'
 3import { immer } from 'zustand/middleware/immer'
 4type CatState = {
 5  cats: {
 6    bigCats: number
 7    smallCats: number
 8  }
 9  increaseBigCat: () => void
10  increaseSmallCat: () => void
11  summary: () => number
12}
13
14const useCatStore = create<CatState>()(
15  immer((set, get) => ({
16    cats: {
17      bigCats: 0,
18      smallCats: 0
19    },
20    increaseBigCat: () =>
21      set(state => {
22        state.cats.bigCats++
23      }),
24    increaseSmallCat: () => set(state => state.cats.smallCats++),
25    summary: () => get().cats.bigCats + get().cats.smallCats
26  }))
27)
28
29export default useCatStore

上述代码中，在传入set的同时，也传入get。在使用时，只需要通过get()来调用对应的state即可。get()可以获取到当前state中所有的内容。

使用选择器（selector）和自动生成选择器

在最开始的基础使用中，每次获取状态或动作，都需要使用useCatStore(state => state.cats.bigCats)这样的形式来获取单个的状态或动作，很繁琐。如果直接使用useCatStore()来获取全部的状态或动作，但只使用其中的一个，又会造成重复渲染。

上述代码中，当在CatBox中操作增加bigCat时，CatBoxReRender中的bigCats会跟随一起变化，随机数也会产生变化。但是在CatBox中操作增加smallCat时，CatBoxReRender中的随机数也会发生变化，造成了重新渲染，这个就不对了。所以就需要写成selector的形式，一条条来调用，但是这种又很繁琐，作者提供了一个auto generate selector的函数，来简化这种调用。

自动生成选择器（auto generating selectors）

新建一个工具函数

 1// src/utils/createSelector.ts
 2import { StoreApi, UseBoundStore } from 'zustand'
 3
 4type WithSelectors<S> = S extends { getState: () => infer T }
 5  ? S & { use: { [K in keyof T]: () => T[K] } }
 6  : never
 7
 8export const createSelectors = <S extends UseBoundStore<StoreApi<object>>>(_store: S) => {
 9  const store = _store as WithSelectors<typeof _store>
10  store.use = {}
11  for (const k of Object.keys(store.getState())) {
12    ;(store.use as any)[k] = () => store(s => s[k as keyof typeof s])
13  }
14
15  return store
16}

然后在store中对create方法进行包裹

 1// src/store/catState-get.ts
 2import { create } from 'zustand'
 3import { createSelectors } from '@/utils/createSelector'
 4import { immer } from 'zustand/middleware/immer'
 5type CatState = {
 6  cats: {
 7    bigCats: number
 8    smallCats: number
 9  }
10  increaseBigCat: () => void
11  increaseSmallCat: () => void
12  summary: () => number
13}
14
15const useCatStore = createSelectors(
16  create<CatState>()(
17    immer((set, get) => ({
18      cats: {
19        bigCats: 0,
20        smallCats: 0
21      },
22      increaseBigCat: () =>
23        set(state => {
24          state.cats.bigCats++
25        }),
26      increaseSmallCat: () => set(state => state.cats.smallCats++),
27      summary: () => get().cats.bigCats + get().cats.smallCats
28    }))
29  )
30)
31
32export default useCatStore

多状态选择器

自动生成选择器还是只能选择第一层的一个状态，如果需要从store中取出多个状态，可以使用下述方法

示例：

 1// src/components/CatBoxController.tsx
 2import useCatStore from '@/store/catState'
 3import { shallow } from 'zustand/shallow'
 4
 5const CatController = () => {
 6  const { increaseBigCat, increaseSmallCat } = useCatStore(
 7    state => ({
 8      increaseBigCat: state.increaseBigCat,
 9      increaseSmallCat: state.increaseSmallCat
10    }),
11    shallow // 无需深拷贝,shallow函数会自动比较第一层的state是否相同，避免重复渲染
12  )
13  return (
14    <div className='box'>
15      <h1 className='title'>Cats Controller</h1>
16      <div>{Math.random()}</div>
17      <div>
18        <button className='btn' onClick={increaseBigCat}>Add Big Cat</button>
19        <button className='btn' onClick={increaseSmallCat}>Add Small Cat</button>
20      </div>
21    </div>
22  )
23}
24
25export default CatController

上述代码，会通过shallow函数对第一层的state进行比较，如果相同则不会重新渲染。所以必须加上shallow，如果有多层嵌套的状态还是需要自己来写一个比较函数。

V5版本修改

在zustand的v5 版本中，修改了shallow的引用，放到了创建store时。

  • 在创建时不使用create来创建store，使用import { createWithEqualityFn } from 'zustand/traditional'来创建store
  • 在createWithEqualityFn的第二个参数中填入shallow，而不需要在引用store时使用shallow。
 1import { createWithEqualityFn } from 'zustand/traditional'
 2import { immer } from 'zustand/middleware/immer'
 3import { shallow } from 'zustand/shallow'
 4
 5const useCateStore = createWithEqualityFn(
 6	immer(
 7  	state => ({
 8    cats: 0,
 9    addCat: () => set(
10    	state => {
11        state.cats++
12      }
13    ),
14    removeCat: set(
15    	state => {
16        state.cats--
17      }
18    )
19  })
20  ),
21  shallow
22)

数据持久化存储 persist

Persist 中间件使您能够将 Zustand 状态存储在存储中（例如 localStorage 、 AsyncStorage 、 IndexedDB 等），从而持久保存其数据。

示例：

 1// src/store/bearState.ts
 2import { create } from 'zustand'
 3import { persist } from 'zustand/middleware'
 4
 5type BearState = {
 6  bears: number
 7  color: string
 8  size: string
 9  increasePopulation: () => void
10  removeAllBears: () => void
11}
12
13const useBearStore = create<BearState>()(
14  persist(
15    set => ({
16      bears: 0,
17      color: 'black',
18      size: 'small',
19      increasePopulation: () => set(state => ({ bears: state.bears + 1 })),
20      removeAllBears: () => set({ bears: 0 })
21    }),
22    {
23      name: 'bears', // name 是必填项
24      // getStorage: () => localStorage // getStorage 可以用来选择存储介质，默认是 localStorage
25      // partialize: state => ({ bears: state.bears })
26      // 排除部分数据
27      partialize: state =>
28        Object.fromEntries(Object.entries(state).filter(([key]) => !['size'].includes(key)))
29    }
30  )
31)
32
33export default useBearStore

上述代码中，使用persist来记录状态到本地的localStorage中，persist的第二个参数是必填项，因为保存到本地时需要一个存储key值，这里需要写上persist的名称。

如果在本地存储中是需要保存部分数据或需要排除部分数据，可以根据上代码中的写法设置parialize属性。

清除本地存储

清除本地存储，可以直接调用useBearStore.persist.clearStorage这个方法。但是需要注意的是，此方法仅清除了localStorage中的数据，并没有清除内存中的数据，所以如果有计数值时，会直接从上次计数开始。此时最好写一个初始化的actions函数来完成状态数据的初始化。

订阅状态 subscribe

直接使用选择器来获取状态，势必会让页面进行重新渲染。当状态每次发生变化都进行重新渲染会造成性能浪费。

示例：

使用subscribe

通过我们可以通过subscribe来限制渲染的时机。

 1// src/components/BearBox.tsx
 2import { useEffect, useState } from 'react'
 3import useBearStore from '@/store/baseState.ts'
 4import { useFoodStore } from '@/store/foodState.ts'
 5
 6const BearBox = () => {
 7  const bears = useBearStore(state => state.bears)
 8  const increasePopulation = useBearStore(state => state.increasePopulation)
 9  const removeAllBears = useBearStore(state => state.removeAllBears)
10
11  const [backgroundColor, setBackgroundColor] = useState<'lightpink' | 'lightgreen'>('lightpink')
12
13  useEffect(() => {
14    const unSub = useFoodStore.subscribe((state, prevState) => {
15      if (state.food > 5 && prevState.food <= 5) {
16        setBackgroundColor('lightgreen')
17      } else if (state.food <= 5 && prevState.food > 5) {
18        setBackgroundColor('lightpink')
19      }
20    })
21
22    return () => unSub()
23  }, [])
24
25  return (
26    <div className='box' style={{ backgroundColor }}>
27      <h1 className='title'>Bears</h1>
28      <p className='py-2'>Bears Count: {bears}</p>
29      <div>{Math.random()}</div>
30      <div>
31        <button className='btn' onClick={increasePopulation}>
32          Add Bear
33        </button>
34        <button className='btn' onClick={removeAllBears}>
35          Remove All Bears
36        </button>
37        <button className='btn' onClick={useBearStore.persist.clearStorage}>
38          Clear Storage
39        </button>
40      </div>
41    </div>
42  )
43}
44
45export default BearBo

上述代码中，可以直接从useFoodStore.subscribe中获取到前次状态和当前状态，通过对两个状态的比较，可以精确控制背景变化的时机。

使用订阅最好放到useEffect中，在加载页面时加载订阅，在使用完后进行销毁。

当需要更多参数来控制订阅加载的时机、对订阅的比较函数进行自定义时，就需要使用subscribeWithSelector这个函数来处理下foodStore。

使用subscribeWithSelector

使用subscribeWithSelector需要对原状态进行包裹，注意包裹层级位置。

 1// src/store/foodState.ts
 2import { create } from 'zustand'
 3import { subscribeWithSelector } from 'zustand/middleware'
 4
 5type FoodState = {
 6  food: number
 7  addFood: () => void
 8  removeFood: () => void
 9  removeAllFood: () => void
10}
11
12export const useFoodStore = create<FoodState>()(
13  subscribeWithSelector((set, get) => ({
14    food: 0,
15    addFood: () => set(() => ({ food: get().food + 1 })),
16    removeFood: () => set(() => ({ food: get().food - 1 })),
17    removeAllFood: () => set({ food: 0 })
18  }))
19)

然后在BearBox中使用，使用方式和直接订阅类似，但参数不一样

 1// src/components/BearBox.tsx
 2import { useEffect, useState } from 'react'
 3import useBearStore from '@/store/baseState.ts'
 4import { useFoodStore } from '@/store/foodState.ts'
 5import { shallow } from 'zustand/shallow'
 6
 7const BearBox = () => {
 8  const bears = useBearStore(state => state.bears)
 9  const increasePopulation = useBearStore(state => state.increasePopulation)
10  const removeAllBears = useBearStore(state => state.removeAllBears)
11
12  const [backgroundColor, setBackgroundColor] = useState<'lightpink' | 'lightgreen'>('lightpink')
13
14  useEffect(() => {
15    const unSub = useFoodStore.subscribe(
16      state => state.food,
17      (food, prevFood) => {
18        if (food > 5 && prevFood <= 5) {
19          setBackgroundColor('lightgreen')
20        } else if (food <= 5 && prevFood > 5) {
21          setBackgroundColor('lightpink')
22        }
23      },
24      {
25        equalityFn: shallow, // 比较函数，用于判断是否变化，可以传入一个自定义的比较函数或使用shallow来使用浅层比较
26        fireImmediately: true // 是否立即执行，如果为false，则只在第一次订阅时执行
27      }
28    )
29
30    return () => unSub()
31  }, [])
32
33  return (
34    <div className='box' style={{ backgroundColor }}>
35      <h1 className='title'>Bears</h1>
36      <p className='py-2'>Bears Count: {bears}</p>
37      <div>{Math.random()}</div>
38      <div>
39        <button className='btn' onClick={increasePopulation}>
40          Add Bear
41        </button>
42        <button className='btn' onClick={removeAllBears}>
43          Remove All Bears
44        </button>
45        <button className='btn' onClick={useBearStore.persist.clearStorage}>
46          Clear Storage
47        </button>
48      </div>
49    </div>
50  )
51}
52
53export default BearBox

上述代码中，useFoodStore.subscribe()的第一个参数是一个选择器，选择对应的状态，第二个参数是一个监听器，可以处理监听前一次和当前状态，第三个参数是一个配置项，equalityFn可以设置比较函数，如果只需要比较一层状态，可以直接使用 shallow。fireImmediately可以设置订阅发生的时机，当加载页面时是否立即执行。否则在第一次订阅时执行。

setState和getState

setState

setState可以更方便的增加一个单独的状态操作函数，而不需要到store中增加对应的action。

getState

getState有别于直接从选择器中获取的状态，从选择器中获取的状态是reactive动态关联的，通过action操作状态，状态会发生变化。而getState的值是non-reactive非动态关联的，也就是说当状态值发生变化时，getState的值不会变化。这样的值有什么用？可以作为其他状态的初始值来使用。

示例：

在没有使用getState设置初始值时，可以在subscribe中设置初始值

 1import { FC, useEffect, useState } from 'react'
 2import useBearStore from '@/store/baseState.ts'
 3import { useFoodStore } from '@/store/foodState.ts'
 4import { shallow } from 'zustand/shallow'
 5

 8  const increasePopulation = useBearStore(state => state.increasePopulation)

10
11  const [backgroundColor, setBackgroundColor] = useState<'lightpink' | 'lightgreen' | undefined>(undefined)
12
13  useEffect(() => {
14    const unSub = useFoodStore.subscribe(
15      state => state.food,
16      (food, prevFood) => {
17        // 如果food和prevFood相等，则根据food设置初始值

56      </div>

使用getState和setState来优化store和selector

在foodState中可以使用setState来设置actrion，在store中仅保留状态变量值，这样就可以将初始值放到state的外部，随时增加修改，而不需要重新定义类型。这样分离state和action可以更简洁的书写代码。

 1// src/store/foodState.ts
 2import { create } from 'zustand'
 3import { devtools, persist, subscribeWithSelector } from 'zustand/middleware'
 4
 5const initialState = {
 6  food: 0
 7}
 8// 使用typeof initialState作为useFoodState的泛型，可以直接根据initialState作为类型，不需要额外定义类型
 9export const useFoodStore = create<typeof initialState>()(
10  devtools(
11    subscribeWithSelector(
12      persist(
13        () => initialState, // 直接使用initialState作为初始状态值
14        {
15        	name: 'food-state'
16      	}
17      )
18    ),
19    {
20      name: 'food-store'
21    }
22  )
23)
24// 所有action都使用setState来定义
25export const addFood = () => useFoodStore.setState(state => ({ food: state.food + 1 }))
26export const removeFood = () => useFoodStore.setState(state => ({ food: state.food - 1 }))
27export const removeAllFood = () => useFoodStore.setState({ food: 0 })

状态切片

当项目越来越大，功能越来越多，store也会越来越庞大，可以将状态划分成更小的store实现模块化。

示例

 1// src/store/bearState.ts
 2import { StateCreator } from 'zustand'
 3
 4export interface BearState {
 5  bears: number
 6  addBear: () => void
 7  removeBears: () => void
 8}
 9
10export const createBearSlice: StateCreator<BearState> = set => ({
11  bears: 0,
12  addBear: () => set(state => ({ bears: state.bears + 1 })),
13  removeBears: () => set(state => ({ bears: state.bears - 1 }))
14})

在ts中需要使用StateCreator来导入类型。

 1// src/store/fishState.ts
 2import { StateCreator } from 'zustand'
 3
 4export interface FishState {
 5  fish: number
 6  addFish: () => void
 7  removeFish: () => void
 8}
 9
10export const createFishSlice: StateCreator<FishState> = set => ({
11  fish: 0,
12  addFish: () => set(state => ({ fish: state.fish + 1 })),
13  removeFish: () => set(state => ({ fish: state.fish - 1 }))
14})

 1// src/store/index.ts
 2import { create } from 'zustand'
 3import { immer } from 'zustand/middleware/immer'
 4import { persist, devtools } from 'zustand/middleware'
 5
 6import { createBearSlice, BearState } from '@/store/bearState'
 7import { createFishSlice, FishState } from '@/store/fishState'
 8
 9const useStore = create<BearState & FishState>()(
10  immer(
11    devtools(
12      persist(
13        (...a) => ({
14          ...createBearSlice(...a),
15          ...createFishSlice(...a)
16        }),
17        {
18          name: 'bear fish store',
19          partialize: state => ({ bears: state.bears })
20        }
21      )
22    )
23  )
24)
25
26export default useStore
