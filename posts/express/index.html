Express 是 Node.js 最受欢迎的 Web 应用框架之一。它提供了简洁而灵活的方式来构建 Web 服务器和 API。Express 提供了一组强大的工具和功能，使得开发 Web 应用变得简单而高效。

express 安装有两种方式：

  • 通过 express 提供的脚手架，直接创建一个项目框架

中间件 middleware

Express是一个路由和中间件组成的web框架，本身的功能非常少。express本质上是一系列中间件函数的调用。

  • 中间件本质是传递给 express 的一个回调函数
  • 回调函数接收三个参数：
    • 请求对象request对象
    • 相应对象response对象
    • next 函数在express中用于执行下一个中间件的函数
  • 中间件可以执行任何代码
  • 更改请求request和响应response对象
  • 结束请求-相应周期（返回数据）
  • 调用栈中的下一个中间件

如果当前中间件功能没有结束请求-响应周期，则必须调用next()函数将控制权传递给下一个中间件，否则，请求将保持挂起状态。

中间件在程序中的使用

express 提供了两种使用方式：

  • app.use或router.use
  • app.method或router.method

最基础的 use 中间件

执行后会在服务端打印出middleware 1，middleware 2，middleware 3，并且在客户端返回Hello World。由于没有匹配任何路径和方法，所以任何路径和方法执行时，在执行next()函数后，use中间件都会被依次执行。

 2const app = express()
 3

 6	console.log('middleware 1')
 7	next()
 8})
 9
10app.use((req, res, next) => {
11	console.log('middleware 2')
12	next()
13})
14
15app.use((req, res, next) => {
16	console.log('middleware 3')
17	res.end('Hello World')
18})
19
20app.listen(7700, () => {
21	console.log('Server is running http://localhost:7700')
22})

匹配路径的中间件

中间件会顺序依次执行，当未匹配到对应的路径时，则不会执行该中间件。由于只匹配了路径，所以任何方法都可以使用到只匹配路径的中间件。

 1const express = require('express')
 2const app = express()
 3
 4app.use((req, res, next) => {
 5	console.log('middleware 1')
 6	next()
 7})
 8
 9// 匹配路径的中间件
10app.use('/home', (req, res, next) => {
11	console.log('middleware home')
12})
13
14app.use('/post', (req, res, next) => {
15	console.log('middleware post')
16	res.end('Hello World')
17})
18
19app.listen(7700, () => {
20	console.log('Server is running http://localhost:7700')
21})
  • 当访问/home时，会依次打印middleware 1、middleware home。但是当执行到 home 中间件时，没有执行next()，也没结束请求-响应周期，所以服务将为挂起状态。
  • 当访问/post时，会依次打印middleware 1、middleware post，并且返回客户端Hello World。

Response 响应数据

将数据返回给客户端，express 主要有：

  • end：res.end（[data] [， encoding]），结束响应，类似于node中的response.end()。

    1res.end('hello')
2res.status(404).end()

  • send：res.send([body])，发送响应并结束响应流程。

    • 参数可以是string、object、butter、boolean或者array。

    • 使用send方法需要配合对应的Content-Type响应头字段设置。

    • send与end的相同点：

      • 二者最终都是回归到 http.ServerResponse.Use 的 response.end() 方法。
      • 二者都会结束当前响应流程。

    • send与end的不同点：

      • end只能发送 string 或者 Buffer 类型，send可以发送任何类型数据。
      • end更适合没有任何响应数据的场景，而send更适合于存在响应数据的场景。
      • 一般建议使用 res.send()方法即可，这样就不需要关心响应数据的格式。
      1// 参数为字符串时
2res.set('Content-Type', 'text/htrml')
3res.send('<p>hello world</p>')
4
5// 参数为数组或对象时
6res.set('Content-Type', 'application/json')
7res.send({ username: 'jacky' })
8res.send([1, 2, 3])

  • json：res.json（[body]）。使用send方法，需要设置对应的响应头来配合显示内容，比较麻烦，所以一般api传递时都是使用json方法，传递数据。

    • 此方法发送一个响应（具有正确的内容类型），该响应是使用 JSON.stringify() 转换为 JSON 字符串的参数。

    • 该参数可以是任何 JSON 类型，包括对象、数组、字符串、布尔值、数字或 null，您还可以使用它将其他值转换为 JSON。

      1res.json(null)
2res.json({ user: 'tobi' })
3res.status(500).json({ error: 'message' })

  • status：res.status(code)，设置响应的 HTTP 状态。

    1res.status(403).end()
2res.status(400).send('Bad Request')

服务端错误

服务端错误可以通过设置错误码和错误信息返回给客户端，通常会通过next()将错误码传递给一个统一的错误中间件来处理，返回对应的状态码和错误内容。

 2const app = express()
 3

 5app.use(express.json())
 6
 7// 使用路由

 9	let isLogin = false
10	const { username, password } = req.body
11
12	// 通过next传递错误信息
13	if (!username || !password) {
14		next(-1001)
15	} else if (username !== 'admin' || password !== '123456') {
16		next(-1002)
17	} else {
18		isLogin = true
