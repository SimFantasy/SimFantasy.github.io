Next.js学习笔记

Simz
2022-06-14
react nextjs

最近在学习Next，这是一个与React相关的应用框架。最近需要使用Next做项目，所以跟着视频一起学了些东西，本文作为笔记进行记录。

因为网络上大多视频，时间比较久远，就导致很多代码的框架、模块的版本发送了很多变化，导致很多功能都发生了很大的变化，当然老的技术你可以学，但是新的还是要学的。比如：最近学了Redux，大多视频讲的都是Redux4.x，结果现在redux官方现在推荐使用ReduxToolkit。东西当然优化了很多，没有原来那么麻烦，但是资料也少啊 ，遇到很多问题，还在摸索。

出现这种情况的还有React，现在网上很多视频打着2022最新教程，讲着React16.13或者17.x的内容，现在React官方已经到了18.x了。看介绍说是优化了很多性能上的东西，但功能也有些调整，还没来得及了解。

列举下最近技术框架发生的变化：

  1. React 当前版本是18.1.0。入口文件的写法调整了，开始学的时候就是一个大坑。不过还好其他的功能变化不是太多。
  2. Redux当前版本是4.2，但是官方建议使用ReduxToolkit，来优化原来的繁琐操作，ReduxToolkit内置了redux-thunk，调整了action和reducer的写法，可以写到一起作为一个状态切片。在store中合并切片导出。
  3. Mobx是一个比Redux好用很多的状态管理库，现在版本是6.6.0，较之前5的写法上优化了很多，可以不适用装饰器，可以通过context传递状态，可以使用makeAutoObservable来对状态进行自动观察，并且可以指定观察类型，并且绑定this。
  4. React-router当前版本是6.3，相较于5的内容，调整了很多写法，之前的react-router-config在6中不好使了，但是可以useRoutes来代替，但某些场景似乎有问题。官方文档就是个渣，搜索都没有。
  5. Next当前版本是12.1.6，好在功能上做了很多优化，功能目前了解到的变化不多。css可以直接引用不需要配置babel，css in js框架内置了styled-jsx，可以实现类似vue种的直接写style，无需引用。使用styled-components，也只需要在next.config.js中配置一下了

好了这里还是说回Next。已下为记录Next功能笔记，所有代码都已亲自敲过并验证。

Next路由

Next使用的是约定式路由模式，在pages目录下存放的每个文件和文件夹都是一个路由，如：/指的是pages下的index.js文件。/blog只的是blog.js或者blog目录下的index.js

在next项目中直接使用a标签进行跳转在跳转过程中，由于是服务端渲染，会加载很多相关依赖的js。这里就需要使用next链接标签Link。

Next路由中的钩子事件

路由钩子事件，也就是当路由发生变化时，可以监听到这些变化的事件，执行对应的函数。如：在routeChangeStart时执行加载动画，在routeChangeComplete时关闭加载动画。

 1import Link from 'next/link'
 2import { Router } from 'next/router'
 3
 4export default function Home() {
 5  //路由发生变化时
 6  Router.events.on('routeChangeStart', (...args) => {
 7    console.log('1. routeChangeComplete: 路由开始变化, 参数为: ',...args)
 8  })
 9  //路由结束变化时
10  Router.events.on('routeChangeComplete', (...args) => {
11    console.log('2. routeChangeComplete: 路由结束变化, 参数为: ',...args)
12  })
13  //浏览器 history触发前
14  Router.events.on('beforeHistoryChange', (...args) => {
15    console.log('3. beforeHistoryChange: 在改变浏览器history之前触发, 参数为: ',...args)
16  })
17  //路由跳转发生错误时
18  Router.events.on('routeChangeError',(...args)=>{
19      console.log('4. routeChangeError: 跳转发生错误, 参数为: ',...args)
20  })
21
22  // hash路由的开始变化
23  Router.events.on('hashChangeStart',(...args)=>{
24    console.log('5. hashChangeStart: hash跳转开始时执行, 参数为: ',...args)
25  })
26  // hash路由的结束变化
27  Router.events.on('hashChangeComplete',(...args)=>{
28      console.log('6. hashChangeComplete: hash跳转完成时, 参数为: ',...args)
29  })
30
31  return (
32    <>
33      <Link href='/user/Jack'><a>history路由跳转</a></Link>
34      <br />
35      <Link href='#/abc'><a>hash路由跳转</a></Link>
36    </>
37  )
38}

Next样式的引用

有四种方式：

  1. 直接引用，/styles/globals.css文件，在index.js中直接进行了引用。此样式一般为全局样式。

  2. 使用模块化引用。/styles/Home.module.css文件在index.js中也进行了模块化引用。

     1// /pages/index.js
 2
 3import styles from '../styles/Home.module.css'
 4
 5export default function Home() {
 6  return (
 7      <>
 8        <main>
 9         {/* 使用模块化样式引用 */}
10        <h2 className={styles.title}>Next首页</h2>
11       </>
12    )
13}

  3. 使用css in js，next中默认自带了styled-jsx，由于是组件内样式，所以在其他组件使用时没有效果。

     1// /pages/index.js
 2
 3export default function Home() {
 4  return (
 5      <>
 6        <main>
 7        <h2 className={styles.title}>Next首页</h2>
 8        {/* 使用styled-jsx样式 */}
 9        <div className='sub-title'>这里是副标题</div>
10        <style>{`
11          .sub-title { color:red; line-height:1.8; font-weight:800; text-align:center; }
12        `}</style>
13       </>
14    )
15}

  4. 还可以使用其他的css in js的技术，比如，styled-components，使用时需先进行安装，然后在next.config.js中进行配置。next新版本不需要在配置babel相关内容，简单很多

    1// 安装
2yarn add styled-components
3
4// 然后在根目录下next.config.js配置这个插件
5const nextConfig = {
6  compiler: {
7    styledComponents: true,
8  },
9}

Next中发起网络请求

使用axios发起网络请求，官方从9.3以上版本建议使用getStaticProps通过context转发数据。以下版本则还可以使用getInitialProps通过props转发数据。两种在写法上有些不同。

 1import React from 'react'
 2import axios from 'axios'
 3
 4const News = (props) => {
 5  const { banners, recommendList } = props
 6  return (
 7    <div>
 8      <h3>News</h3>
 9      <ul>
10        {banners.map(item => (<li key={item.amc}>{item.title}</li>))}
11      </ul>
12      <hr />
13      <ul>
14        {recommendList.map(item => (<li key={item.amc}>{item.title}</li>))}
15      </ul>
16    </div>
17  )
18}
19// 此处需要将getStaticProps进行导出，否则报错，并且getStaticProps的形参须为context，return的结构为返回一个对象中的props对象
20export async function getStaticProps(context) {
21  const res = await axios.get('http://123.207.32.32:8000/home/multidata').then(res => res.data)
22  return {
23    props: {
24      banners: res.data.banner.list,
25      recommendList: res.data.recommend.list
26    }
27  }
28}
29
30export default News

 2
 3export default function Time() {
