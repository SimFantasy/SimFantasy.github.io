Node 10 ORM Sequelize

Simz
2023-10-08
node sequelize

Sequelize 是一个基于 promise 的 Node.js ORM，目前支持 Postgres、MySQL、MariaDB、 SQLite 以及 Microsoft SQL Server。它具有强大的事务支持，关联关系, 预读和延迟加载,读取复制等功能。

Sequelize 遵从 语义版本控制。 支持 Node v10 及更高版本以便使用 ES6 功能。

安装

1pnpm add mysql2 sequelize

mysql2为MySQL数据驱动。

初始化

连接数据库

定义模型

模型是 Sequelize 的本质。模型是代表数据库中表的抽象，在 Sequelize 中，它是一个 Model 的扩展类。

该模型告诉 Sequelize 有关它代表的实体的几件事，例如数据库中表的名称以及它具有的列(及其数据类型).

Sequelize 中的模型有一个名称，此名称不必与它在数据库中表示的表的名称相同，通常模型具有单数名称(例如,User)，而表具有复数名称(例如, Users)，这些可以通过配置进行修改。

同步模型

定义模型时,你要告诉 Sequelize 有关数据库中表的一些信息.。但是，如果该表实际上不存在于数据库中怎么办？ 如果存在,但具有不同的列,较少的列或任何其他差异,该怎么办？

这就是模型同步的来源。可以通过调用一个异步函数(返回一个Promise)model.sync(options).。通过此调用,Sequelize 将自动对数据库执行 SQL 查询.。请注意,这仅更改数据库中的表，而不更改 JavaScript 端的模型。

  • User.sync() - 如果表不存在,则创建该表(如果已经存在,则不执行任何操作)
  • User.sync({ force: true }) - 将创建表，如果表已经存在,则将其首先删除
  • User.sync({ alter: true }) - 这将检查数据库中表的当前状态(它具有哪些列，它们的数据类型等)，然后在表中进行必要的更改以使其与模型匹配。
 1const sequelize = require('../db/config')
 2
 3require('./models')
 4
 5// 同步数据库
 6sequelize
 7	.sync({ alter: true })
 8	.then(() => {
 9		console.log('所有模型同步完成')
10	})
11	.catch(err => {
12		console.log('模型同步失败', err)
13	})

在package.json中修改scripts：

1"scripts": {
2  "dev": "nodemon src/main.js",
3  "sync": "node src/model/sync.js"
4},

时间戳

默认情况下，Sequelize 使用数据类型 DataTypes.DATE 自动向每个模型添加 createdAt 和 updatedAt 字段。这些字段会自动进行管理，每当你使用Sequelize 创建或更新内容时，这些字段都会被自动设置。 createdAt 字段将包含代表创建时刻的时间戳，而 updatedAt 字段将包含最新更新的时间戳。

如果不需要显示这两个时间戳，可以在模型中设置timestamps: false，则表示不需要显示时间戳。

数据操作增删改查

添加数据

添加数据有两种方法，build和create。通常情况下使用create直接创建实例即可。

build方法示例：

 1const { Admin } = require('../model/models')
 2
 3const createAdmin = async (adminData) => {
 4  // 同步方法，构建一个模型实例
 5  const instence = Admin.build(adminData)
 6  // 在保存之前还可以对数据进行修改
 7  instence.password = handleMd5(instence.password)
 8  // 最后需要保存到数据库，是个异步方法
 9  const res = await instence.save()
10  return res
11}

create方法示例：

1const { Admin } = require('../model/models')
2
3const createAdmin = async (adminData) => {
4  const result = await Admin.create(adminData)
5  return result
6}

批量创建

Sequelize 提供了 Model.bulkCreate 方法，以允许仅一次查询即可一次创建多个记录。通过接收一个数组对象来创建多条数据。

1const captains = Captain.bulkCreate([
2  { name: 'Jack Sparrow' },
3  { name: 'Davy Jones' }
4]);

默认情况下，bulkCreate 不会在要创建的每个对象上运行验证(而 create 可以做到)。为了使 bulkCreate 也运行这些验证，必须通过validate: true 参数，但这会降低性能。

 1// 定义模型
 2const Foo = sequelize.define('foo', {
 3  bar: {
 4    type: DataTypes.TEXT,
 5    validate: {
 6      len: [4, 6]
 7    }
 8  }
 9});
10
11// 批量创建
12// 没有设置：validate: true，这不会引发错误,两个实例都将被创建
13await Foo.bulkCreate([
14  { name: 'abc123' },
15  { name: 'name too long' }
16]);
17
18// 设置：validate: true，这将引发错误,不会创建任何内容
19await Foo.bulkCreate([
20  { name: 'abc123' },
21  { name: 'name too long' }
22], { validate: true });

45      [Op.match]: Sequelize.fn('to_tsquery', 'fat & rat') // 匹配文本搜索字符串 'fat' 和 'rat' (仅 PG)
46
47      // 在 Postgres 中, Op.like/Op.iLike/Op.notLike 可以结合 Op.any 使用:
48      [Op.like]: { [Op.any]: ['cat', 'hat'] }  // LIKE ANY ARRAY['cat', 'hat']
49    }
50  }
51});

软删除paranoid

Sequelize 支持 paranoid 表的概念。 一个 paranoid 表是一个被告知删除记录时不会真正删除它的表。反而一个名为 deletedAt 的特殊列会将其值设置为该删除请求的时间戳。

关联

Sequelize 支持标准关联关系：一对一、 一对多 和 多对多。

Sequelize 使用四种关联类型进行模型关联。

  • HasOne：有一个关联模型，A模型有一个关联的B模型
  • BelongsTo：属于一个关联模型，A模型属于B模型
  • HasMary：有多个关联模型，A模型有多个B模型
  • BelongsToMary：A模型属于多个B模型，B模型也数据多个A模型。
1A.hasOne(B, { /* 参数 */ });
2A.belongsTo(B, { /* 参数 */ });
3A.hasMany(B, { /* 参数 */ });
4A.belongsToMany(B, { through: 'C', /* 参数 */ });

关联的定义顺序很重要，上的解释中，A模型是源模型，B模型是目标模型。

A.hasOne(B) 关联意味着 A 和 B 之间存在一对一的关系，外键在目标模型(B)中定义。

A.belongsTo(B)关联意味着 A 和 B 之间存在一对一的关系，外键在源模型中定义(A)。

A.hasMany(B) 关联意味着 A 和 B 之间存在一对多关系，外键在目标模型(B)中定义。

A.belongsToMany(B, { through: 'C' }) 关联意味着将表 C 用作联结表,在 A 和 B 之间存在多对多关系，具有外键(例如：aId 和 bId)。 Sequelize 将自动创建此模型 C(除非已经存在)，并在其上定义适当的外键。

一对一关联关系比较简单，此处不做过多说明。

一对多

一对多关联将一个源与多个目标连接，而所有这些目标仅与此单个源连接。

在上面的模型示例中，班级和学生就是一个一对多的关系，一个班级有多个学生，一个学生属于一个班级。所以在模型中设置的关联关系如下：

 1// Student与Class关联关系
 2// 学生属于班级
 3Student.belongsTo(Class, {
 4	foreignKey: 'classId',
 5	targetKey: 'id',
 6	as: 'class'
 7})
 8// 班级有多个学生
 9Class.hasMany(Student, {
10	foreignKey: 'classId',
11	sourceKey: 'id',
12	as: 'students'
13})

在学生模型中，我们使用classId将学生与班级进行关联。

参数：

在相对关系中对应模型所表示的外键也不同。

在belongsTo关系中：

foreignKey：Student模型中对应的外键字段classId。

targetKey：Class模型中对应的目标字段id。

在hasMary关系中：

foreignKey：表示的是Student中对应的classId外键字段。

sourceKey：表示Class自己模型中的id源字段。

as：为模型的关联时的别名。

多对多

对多关联将一个源与多个目标相连，而所有这些目标又可以与第一个目标之外的其他源相连。

不能像其他关系那样通过向其中一个表添加一个外键来表示这一点.。取而代之的是使用联结模型的概念。这将是一个额外的模型(以及数据库中的额外表)。它将具有两个外键列并跟踪关联。

在上的模型示例中，学生和书籍就是多对多的关系

 1// Student与Book关联关系
 2Student.belongsToMany(Book, {
 3	through: StudentBook,
 4	foreignKey: 'studentId',
 5	otherKey: 'bookId',
 6	as: 'books'
 7})
 8
 9Book.belongsToMany(Student, {
10	through: StudentBook,
11	foreignKey: 'bookId',
12	otherKey: 'studentId',
13	as: 'students'
14})

学生和书籍的关系是多对多的关联，所以他们需要一张第三方表进行关联，记录学生Id和学生书籍的Id，也就是在示例中对应的StudentBook模型。

参数

through：belongsToMary关联关系中的第三方关联模型StudentBook。

foreignKey：

  • 在Student与StudentBook模型中关联的字段是studentId。
  • 在Book与StudentBook模型中关联的字段是bookId。

otherKey：则与foreignKey相反，标示该模型在第三方模型中外键需要对应的字段。

  • 在Student模型在StudentBook模型中外键studentId需要对应的是bookId字段
  • 在Book模型在StudentBook模型中外键bookId需要对应的是studentId字段

预先加载

如关联指南中简要提到的,预先加载是一次查询多个模型(一个"主"模型和一个或多个关联模型)的数据的行为。在 SQL 级别上,这是具有一个或多个 join 的查询。

完成此操作后，Sequelize 将在返回的对象中将适当关联的模型添加到适当命名的自动创建的字段中。

在定义模型示例中，我们已经做好了模型上的外键关联，那么在查询时，如果需要将关联的外键关模型中的数据也一起查询出来，就需要用到这个预先加载。

在 Sequelize 中，主要通过在模型查找器查询中使用 include 参数(例如：findOne、findAll 等)来完成预先加载。

示例：

 1const { Class, Student } = require('../model/models')
 2
 3const classList = async (limit = 10, offset = 0) => {
 4  const result = await Class.findAll({
 5    attributes: ['id', 'name', 'open_date'],
 6    limit: Number(limit),
 7    offset: Number(offset),
 8    include: {
 9      model: Student,
10      as: 'students',
11      attributes: ['id', 'name', 'mobile'],
12      required: false
13    }
14  })
15  return result
16}

预先加载使用include参数来设置对应预先加载的关联模型

参数：

model：这就是需要关联的模型名称，需要在此处导入改模型。

as：模型的别名，这个是查询返回数据中显示的字段名称，如果没有设置默认就返回模型的别名。

attributes：也可以来限制关联字段显示的内容。

required：预先加载时，可以强制查询仅返回具有关联模型的记录，从而有效地将查询从默认的 OUTER JOIN 转为 INNER JOIN，这是通过设置 required: true

多次预先加载

include可以匹配多个形式的模型预先加载

 1// 直接加载模型
 2const classList = await Task.findAll({ include: Student });
 3// 加载单个模型配置
 4const classList = await Task.findAll({ 
 5  include: {
 6    model: Student,
 7    as: 'students',
 8    attributes: ['id', 'name', 'mobile'],
 9    required: false
10  }
11});
