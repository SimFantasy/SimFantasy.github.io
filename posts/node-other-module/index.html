MySQL2

在Node项目中连接MySQL可以使用MySQL2库。MySQL2 大部分 API 与 mysqljs 兼容，并支持大部分功能。 MySQL2 还提供了更多的附加功能：

  • 更快、更好的性能
  • 支持预处理
  • MySQL二进制日志协议
  • MySQL Server
  • 对编码和排序规则有很好的支持
  • Promise封装
  • 支持压缩
  • SSL 和 Authentication Switch
  • 自定义流
  • 连接池

官方文档地址：MySQL2中文文档

安装

1pnpm add mysql2

基本使用

  • 创建连接（通过createConnection），并且获取连接对象。
  • 执行SQL语句即可（通过query）

 2

预处理语句

  • 提高性能：将创建的语句模块发送给MySQL，然后MySQL编译（解析、优化、转换）语句模块，并且存储它但是不执行，之后在真正执行时会给?提供实际的参数才会执行，就算多次执行，也只会编译一次，所以能提高性能。
  • 防止SQL注入：之后传入的值不会像模块引擎那样就编译，那么一些SQL注入的内容不会被执行。
  • 如果再次执行该语句，它将会从LRU（Least Recently Used） Cache中获取获取，省略了编译statement的时间来提高性能。

连接池

连接池通过重用以前的连接来帮助减少连接到 MySQL 服务器所花费的时间，当完成它们时让它们保持打开而不是关闭。这改善了查询的延迟，避免了建立新连接所带来的所有开销。

  • 连接池可以在需要的时候自动创建连接，并且创建的连接不会被销毁，会放到连接池中，后续可以继续使用。
  • 以在创建连接池的时候设置LIMIT，也就是最大创建个数
 1const mysql = require('mysql2')
 2
 3// 创建连接
 4const pool = mysql.createPool({
 5	host: 'localhost',
 6	port: 3306,
 7	database: 'db_school',
 8	user: 'root',
 9	password: '123456',
10	connectionLimit: 10
11})
12
13// 从连接池中获取连接
14pool.getConnection((err, conn) => {
15	conn.connect(err => {
16		if (err) {
17			console.log(err)
18		} else {
19			console.log('conn success')
20		}
21	})
22})
23
24const statement = 'select * from mobiles where price > ? and brand = ?'
25pool.execute(statement, [1000, '华为'], (err, result, fields) => {
26	if (err) {
27		console.log(err)
28	} else {
29		console.log('result', result)
30		console.log('fields', fields)
31	}
32})

 2const Koa = require('koa')

 9const pool = mysql.createPool({

13	user: 'root',
14	password: '123456',
15	connectionLimit: 10
16})
17
18// 从连接池中获取连接
19pool.getConnection((err, conn) => {
20	conn.connect(err => {
21		if (err) {
22			console.log(err)
23		} else {
24			console.log('conn success')
25		}
26	})
27})
28
29// 通过Promise的方式获取数据，将数据返回给页面
30router.get('/', async ctx => {
31	const statement = 'select * from mobiles where price > ? and brand = ?'
32	const [result] = await pool.promise().execute(statement, [1000, '华为'])
33
34	ctx.body = result
35})
36
37app.use(router.routes(), router.allowedMethods())
38app.listen(3000, () => {

40})

  • rows，会返回查询表的结构。

Cookie

Cookie（复数形态Cookies），又称为“小甜饼”。类型为“小型文本文件，某些网站为了辨别用户身份而存储 在用户本地终端（Client Side）上的数据。

  • 浏览器会在特定的情况下携带上Cookie来发送请求，可以通过Cookie来获取一些信息
  • Cookie总是保存在客户端中，按在客户端中的存储位置，Cookie可以分为内存Cookie和硬盘Cookie。
    • 内存Cookie由浏览器维护，保存在内存中，浏览器关闭时Cookie就会消失，其存在时间是短暂的。
    • 硬盘Cookie保存在硬盘中，有一个过期时间，用户手动清理或者过期时间到时，才会被清理。
    • 没有设置过期时间，默认情况下Cookie是内存Cookie，在关闭浏览器时会自动删除。
    • 有设置过期时间，并且过期时间不为0或者负数的Cookie，是硬盘Cookie，需要手动或者到期时，才会删除。

Cookie配置

name：key的名称

value：Cookie的值

maxAge：过期时间。如果只想设置 Cookie 的保存路径而不想设置存活时间。

  • MaxAge=0意味着没有指定Max-Age属性。
  • MaxAge<0意味着现在就删除Cookie，等同于 Max-Age: 0。设置Cookie为-1即可。
  • MaxAge>0意味着存在Max-Age属性，并以秒为单位。

path：Cookie的路径

domain：Cookie 的路径Domain作用域本地调试配置成localhost , 正式上线配置成域名。

secure：当 secure 值为 true时，Cookie 在 HTTP 中是无效，在 HTTPS 中才有效。

httpOnly：是微软对 Cookie 做的扩展。如果在 Cookie 中设置了httpOnly属性， 则通过程序（JS 脚本、applet 等）将无法读取到 Cookie 信息，防止 XSS 攻击产生。

Cookie的生命周期

默认情况下的Cookie是内存Cookie，也称之为会话Cookie，也就是在浏览器关闭时会自动被删除。可以通过设置expires或者max-age来设置过期的时间。

  • expires：设置的是Date.toUTCString()，设置格式是：expires=date-in-GMTString-format。
  • max-age：设置过期的时间，单位为秒s，max-age=max-age-in-seconds (例如一年：60*60*24*365)；

Cookie的作用域

允许Cookie发送给哪些URL。

  • Domain：指定哪些主机可以接受Cookie。

    • 如果不指定，那么默认是 origin，不包括子域名。
    • 如果指定Domain，则包含子域名。例如，如果设置 Domain=mozilla.org，则Cookie 也包含在子域名中（如developer.mozilla.org）。

  • Path：指定主机下哪些路径可以接受Cookie。例如，设置 Path=/docs，则以下地址都会匹配：

    1/docs
2/docs/Web/
3/docs/Web/HTTP

Session

session 是另一种记录客户状态的机制，不同的是 Cookie 保存在客户端浏览器中，而 session 保存在服务器上。

Session的工作流程

当客户端浏览器第一次访问服务器并发送请求时，服务器端会创建一个 session 对象，生成 一个类似于 key,value 的键值对，然后将 value 保存到服务器 将 key(cookie)返回到浏览器(客 户)端。浏览器下次访问时会携带 key(cookie)，找到对应的 session(value)。

Koa中的Session

Token

Cookie和Session的方式有很多的缺点：

  • Cookie会被附加在每个HTTP请求中，所以无形中增加了流量（事实上某些请求是不需要的）
  • Cookie是明文传递的，所以存在安全性的问题
  • Cookie的大小限制是4KB，对于复杂的需求来说是不够的
  • 对于浏览器外的其他客户端（比如iOS、Android），必须手动的设置Cookie和Session
  • 对于分布式系统和服务器集群中其他系统也可以正确的解析Session会很麻烦

在目前的前后端分离的开发过程中，使用Token来进行身份验证的是最多的情况：

  • token可以翻译为令牌
  • 也就是在验证了用户账号和密码正确的情况，给用户颁发一个令牌
  • 这个令牌作为后续用户访问一些接口或者资源的凭证
  • 可以根据这个凭证来判断用户是否有权限来访问

token的使用应该分成两个重要的步骤

  • 生成token：登录的时候，颁发token
  • 验证token：访问某些资源或者接口时，验证token

JWT

JSON Web Token（JSON Web令牌）。是一个开放标准(rfc7519)，它定义了一种紧凑的、自包含的方式，用于在各方之间以JSON对象安全地传输信息。此信息可以验证和信任，因为它是数字签名的。JWT可以使用秘密〈使用HNAC算法）或使用RSA或ECDSA的公钥/私钥对进行签名。通过JSON形式作为Web应用中的令牌，用于在各方之间安全地将信息作为JSON对象传输。在数据传输过程中还可以完成数据加密、签名等相关处理。

JWT作用

  • 授权：一旦用户登录，每个后续请求将包括JWT，从而允许用户访问该令牌允许的路由，服务和资源。它的开销很小并且可以在不同的域中使用。如：单点登录。
  • 信息交换：在各方之间安全地传输信息。JWT可进行签名（如使用公钥/私钥对)，因此可确保发件人。由于签名是使用标头和有效负载计算的，因此还可验证内容是否被篡改。

JWT实现Token机制

JWT生成的Token组成

  • header
    • alg：采用的加密算法，默认是 HMAC SHA256（HS256），采用同一个密钥进行加密和解密
    • typ：JWT，固定值，通常都写成JWT即可
    • 通过base64Url算法进行编码
  • payload
    • 携带的数据，比如可以将用户的id和name放到payload中
    • 默认也会携带iat（issued at），令牌的签发时间
    • 也可以设置过期时间：exp（expiration time）
    • 通过base64Url算法进行编码
  • verify signature
    • 设置一个secretKey，通过将前两个的结果合并后进行HMACSHA256的算法
    • HMACSHA256(base64Url(header)+.+base64Url(payload), secretKey)
    • 如果secretKey暴露是一件非常危险的事情，因为之后就可以模拟颁发token，也可以解密token；

非对称加密

使用HS256加密算法单一密钥暴露就是非常危险的事，因为：

  • 在分布式系统中，每一个子系统都需要获取到密钥
  • 拿到这个密钥后这个子系统既可以发布另外，也可以验证令牌
  • 是对于一些资源服务器来说，它们只需要有验证令牌的能力就可以了

所以，这个时候可以使用非对称加密，RS256：

  • 私钥（private key）：用于发布令牌
  • 公钥（public key）：用于验证令牌

可以使用openssl来生成一对私钥和公钥：

  • Mac直接使用terminal终端即可
  • Windows默认的cmd终端是不能直接使用的，建议直接使用git bash终端
1openssl
2 genrsa -out private.key 2048
3 rsa -in private.key -pubout -out public.key

注意：RS256生成私钥最小需要2048bit，否则会生成token失败

 1const fs = require('fs')
 2const Koa = require('koa')
 3const Router = require('@koa/router')
 4const jwt = require('jsonwebtoken')
 5
 6const app = new Koa()
 7const router = new Router()
 8
 9// 生成密钥对
10const PrivateKey = fs.readFileSync('./key/private.key')
11const PublicKey = fs.readFileSync('./key/public.key')

15		id: 1,
16		name: 'simz'
17	}
18
19	// 使用私钥生成token
20	const token = jwt.sign(user, PrivateKey, {
21		expiresIn: '1h', // 有效期

23	})
24
25	ctx.body = {

42				// 指定算法

44			})
45			ctx.body = result
46		} catch (error) {
47			ctx.body = 'token verify fail'
48		}
49	} else {
50		// token不存在，则提示token不存在

52	}
53})

59})

Md5加密

在保存密码时， 我们通常会对密码进行加密，当用户需要验证密码时，我们也会把用户输入的密码加密后与数据库中的密码进行比较。

koa中密码进行md5加密使用node自带的crypto标准库。通常可以写成一个函数用于加密。

 1const crypto = require('crypto')
 2
 3// 处理密码
 4const handlePassword = password => {

错误处理

有多种处理方式，可以使用koa中间件来将koa的错误处理抛出。此方法无法捕获404错误。

 1// src/middleware/error.middleware.js
 2const ErrorMiddleware = async (ctx, next) => {
 3	try {
 4		await next()
 5	} catch (error) {
 6		ctx.body = {
 7			code: error.status || error.statusCode || 500,
 8			msg: error.message || '服务器错误'

10	}
11}
12
13module.exports = ErrorMiddleware

koa-json-error

也可以使用koa-json-err来处理错误信息，这个中间件不单可以覆盖到所有的koa错误，包括404错误，也可以输出错误的堆栈信息和错误代码。

安装

1pnpm add koa-json-error

使用

在入口文件中导入并使用中间件，最好在其他中间件之前。以便捕获错误。

 1// src/app/index.js
 2const Koa = require('koa')
 3const bodyparser = require('koa-bodyparser')
 4const useRoutes = require('../router')
 5const error = require('koa-json-error')
 6
 7
 8const app = new Koa()
 9// 配置koa-json-error错误中间件，并配置在生成环境不输出stack堆栈错误信息，只在开发环境输出stack
10app.use(
11	error({
12		postFormat: (e, { stack, ...rest }) =>
13			process.env.NODE_ENV === 'production' ? rest : { stack, ...rest }
14		// postFormat: (e, { stack, ...rest }) => rest
15	})
16)
17
18app.use(bodyparser())
19
20useRoutes(app)
21
22module.exports = app

使用方式同上面自定义，可以手动抛出错误，会自动捕获错误并获取错误消息。

art template是国人开发的一款模板引擎，渲染速度基本上是ejs的五倍多。功能上也比ejs强大更多。

Koa第三方中间件

module-alias

使用module-alias我们就可以轻松的配置使用路径别名，从此和const userData = require('../../file/user.json')这种路径说拜拜。 我们只需要使用const userData = require("@/file/user.json")这种引入方式引入数据，看起来结构更清晰。

安装
1pnpm add module-alias
使用

将自定义的路径别名添加在package.json中

1"_moduleAliases": {
2"@": "./src",
3"@app": "./src/app"
4}

然后在应用的主文件中添加此行，在任何代码之前。

1require('module-alias/register')

然后开始使用吧。示例：

1require('@app/db')
2const module = require('@/router')

注意： 该插件在sequelize中同步数据库时有问题，需要恢复部分sequelize使用到数据库的引用路径。

require-directory

通过递归遍历指定目录下的每个文件，自动加载目录下的路由文件内容。

可以用该方法来替换自己写的遍历路由下的模块。

 1// src/router/index.js
 2const fs = require('fs')
 3
 4const useRoutes = app => {
 5	const files = fs.readdirSync(__dirname)
 6
 7	for (let file of files) {
 8		if (file === 'index.js') continue
 9
10		const router = require(`./${file}`)
11		app.use(router.routes())
12		app.use(router.allowedMethods())
13	}
14}
15
16module.exports = useRoutes
安装
1pnpm add require-directory
使用
 1// src/app/index.js
 2const Router = require('@koa/router')
 3const requireDirectory = require('require-directory')
 4...
 5app.use(bodyparser())
 6// 自动加载路由
 7// 第一个参数module，读取路由目录下的每个模块文件
 8// 第二个参数为路由目录的地址，默认为：../routes，如需修改可以传入一个绝对路径
 9// 第三个参数visit，可以在加载对象时访问对象,判断是否为Router类型对象并加载routes和allowedMethods
10requireDirectory(module, path.resolve(__dirname, '../router'), {
11	visit: obj => {
12		if (obj instanceof Router) {
13			app.use(obj.routes()).use(obj.allowedMethods())
14		}
15	},
16	// 排除router下的index.js文件
17	exclude: /\/index.js/
18})
19
20...
