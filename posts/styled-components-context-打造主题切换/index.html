Styled Components + Context 打造主题切换

Simz
2022-07-29
react styled-components

这两天在看Mantine，里面有自带的主题切换，试了下，感觉过于复杂context的Privoder都要包两层，另外Mantine的css in js 方案使用的是emotion，过于小众和繁琐，不如在styled-components中写css的流畅。所以翻了很久掘金挖宝，果然挖到一宝。Up是椒盐味花生米，使用context修改styled-components的GlobalStyle，并且将配色方案保存在localstorage中，重启浏览器效果依然得以保存。但是他的方案使用的是redux toolkit，并且只能切换明亮和暗黑两种主题，我改成了mobx和多主题，本来想用更轻量的Jotai状态库来试下的，但是Jotai还没摸透，等下次吧。

目录结构：

 1/src
 2	/store
 3		/index.jsx					// 合并状态并存到context中
 4		/theme.store.jsx 		//保存主题状态
 5	/styles
 6		/themes							// 主题文件夹
 7			/index.jsx				// 将主题装入context
 8			/blue.theme.jsx		// 蓝色主题方案
 9			/dark.theme.jsx		// 暗黑主题方案
10			/light.theme.jsx	// 明亮主题方案 
11		/global.style.jsx		// styled-components 系统默认样式
12	/pages
13		/home.jsx						// 展示页面
14	/App.jsx							// 根组件

Mobx实现store

将主题状态存入store，便于在多个页面进行切换设置。

 1// /src/store/theme.store.jsx
 2
 3import { makeAutoObservable } from 'mobx'
 4// 获取localstorage中的主题
 5const localTheme = localStorage.getItem('theme')
 6
 7class ThemeStore {
 8  theme = localTheme ? localTheme : 'light' // 如果localstorage中有主题则使用，否则使用明亮主题
 9  constructor() {
10    makeAutoObservable(this)
11  }
12	// 设置主题
13  setTheme = (data) => {
14    this.theme = data
15  }
16  // 修改主题
17  changeTheme = (state) => {
18    this.setTheme(state)									// 将得到的主题存入store
19    localStorage.setItem('theme', state)	// 将得到的主题存入localstorage
20  }
21}
22
23export default ThemeStore

让后将theme.store.jsx文件中的ThemeStore放入store/index.jsx合并并包装入context，这里其实跟redux toolkit有点像，他是将theme.slice放入index文件中进行合并导出。

 1// /scr/store/index.jsx
 2
 3import { useContext, createContext } from 'react'
 4import ThemeStore from './theme.store'
 5
 6class RootStore {
 7  constructor() {
 8    this.themeStore = new ThemeStore()
 9  }
10}
11
12const rootStore = new RootStore()
13const context = createContext(rootStore)
14const useStore = () => useContext(context)
15
16export default useStore

再来关键的一个封装，将styled-components的主题支持ThemeProvider封装。然后再将封装好的Provider 去包装根组件。

 1// /src/styles/themes/index.jsx
 2
 3import React from 'react'
 4import { observer } from 'mobx-react-lite'
 5import { ThemeProvider } from 'styled-components'
 6import useStore from '@/store'
 7
 8import darkTheme from './dark.theme'
 9import lightTheme from './light.theme'
10import blueTheme from './blue.theme'
11
12
13const ThemeProviderWrapper = (props) => {
14  const {themeStore: { theme }} = useStore() // 从store中获取主题
15  const switchTheme = () => { // 将获取到的主题来匹配对应的主题方案，并return出来给到ThemeProvider
16    switch (theme) {
17      case 'dark':
18        return darkTheme
19      case 'light':
20        return lightTheme
21      case 'blue':
22        return blueTheme
23      default:
24        return darkTheme
25    }
26  }
27  return (
28    // <ThemeProvider theme={theme === 'dark' ? darkTheme : lightTheme} {...props}> // 如果只有两个主题这个更简单
29    <ThemeProvider theme={switchTheme} {...props}>
30      {theme}
31      {props.children}
32    </ThemeProvider>
33  )
34}
35// 因为有用到mobx所以记得这里要用observer来包裹下
36export default observer(ThemeProviderWrapper)

然后再使用styled-components设置全局样式GlobalStyles

 1import { createGlobalStyle } from 'styled-components'
 2
 3const GlobalStyle = createGlobalStyle`
 4  body {
 5    background-color:${(props) => props.theme.colors.body};
 6  }
 7  .header {
 8    background-color:${(props) => props.theme.header.backgroundColor};
 9  }
10`
11export default GlobalStyle

如何使用

首先将封装的ThemeProviderWrapper包裹下根组件，并且将GlobalStyles也包裹进其中，让其生效。

 1import React from 'react'
 2import { BrowserRouter } from 'react-router-dom'
 3import RootRouter from '@/router'
 4import ThemeProviderWrapper from '@/styles/themes'
 5import GlobalStyle from '@/styles/global.style'
 6
 7export default function App() {
 8  return (
 9    <BrowserRouter>
10      <ThemeProviderWrapper>
11        <GlobalStyle />
12        <RootRouter />
13      </ThemeProviderWrapper>
14    </BrowserRouter>
15  )
16}

如果需要在同一个按钮上切换，写个方法即可。

 1// /src/pages/home.jsx
 2
 3import React from 'react'
 4import { observer } from 'mobx-react-lite'
 5import useStore from '@/store'
 6
 7function Rank() {
 8  const { themeStore } = useStore()
 9  const { theme, changeTheme } = themeStore
10
11  return (
12    <div>
13      <div>Rank</div>
14      <div>{theme}</div>
15      <button onClick={() => changeTheme('light')}>change light theme</button>
16      <button onClick={() => changeTheme('dark')}>change dark theme</button>
17      <button onClick={() => changeTheme('blue')}>change blue theme</button>
18      <div className="header" style={{ width: '100%', height: '56px' }}></div>
19    </div>
20  )
21}
22
23export default observer(Rank)

总结

store使用mobx感觉还是有些过重，并且每次都需要将使用mobx的组件页面使用observer进行包裹很麻烦，有时候总是忘记，调了半天才想起来是没进行包裹，mobx无法产生响应式观察。后续Jotai研究出来了，在来替换吧。
