React Hook Form

Simz
2024-02-22

react-hook-form是一个表单管理、验证库。之前有使用过formik，模板代码过多，效率也较差，所以一直有关注这块的库，现在表单库中react-hook-form的知名度无疑是最高的，相比阿里的formly学习成本也低很多。

参考大神 Vishwas 的油管视频 React Hook Form Tutorials 教程来入个门

环境准备

使用vite和typescript来搭建开发环境，在之前的文章中已经分享过如何配置React中ts环境搭建，此处不在赘述。

安装

1# 安装react hook form
2pnpm add react-hook-form

基础使用

示例

首先在src目录下新建components目录，添加base-form.tsx文件。并在App.tsx中引用该文件。

 1// src/components/base-form.tsx
 2import { FC } form 'react'
 3import { useForm } from 'react-hook-form'
 4
 5// 定义表单字段类型
 6type FormData = {
 7  username: string
 8  email: string
 9  channel: string
10}
11
12const BaseForm: FC = () => {
13  // 从hooks中解构出对应的方法
14  const { 
15    register, 
16    handleSubmit
17  } = useForm<FormData>({
18    // 设置表单初始值
19    defaultValues: {
20      username: '',
21      email: '',
22      channel: ''
23    }
24  })
25  
26  const onSubmit = data => console.log('form data: ', data)
27  
28  return (
29  	<div>
30    	<h1>Base Form</h1>
31      <form onSubmit={handleSubmit(onSubmit)} noValidate>
32      	<div className='form-control'>
33        	<label htmlfor='username'>Username</label>
34          <input type='text' id='username' {...register('username')} />
35        </div>
36        
37        <div className='form-control'>
38        	<label htmlfor='email'>Email</label>
39          <input type='text' id='email' {...register('email')} />
40        </div>
41        
42        <div className='form-control'>
43        	<label htmlfor='channel'>Channel</label>
44          <input type='text' id='channel' {...register('channel')} />
45        </div>
46        
47        <button type='submit'>Submit</button>
48      </form>
49    </div>
50  )
51}
52
53export default BaseForm

上述代码中，{...register('username')}的书写方式是ref写法的一种语法糖。

 1const { onChange, onBlur, name, ref} = register('username')
 2
 3<input 
 4	type='text'
 5	onChange={onChange} // 绑定onChange事件
 6	onBlur={onBlur} // 绑定onBlur事件
 7	name={name} // 绑定name属性
 8	ref={ref} // 绑定ref属性
 9/>
10
11// ref语法糖写法
12<input type='text' {...register('username')} />

当点击Submit按钮后，会将表单数据通过onSubmit回调函数中的data参数返回。

使用步骤

  1. 通过useForm结构出对应的方法和对象
  2. 因为使用的Typescript所以，需要定义字段类型 FormData，并将类型通过泛型的方式给到useForm，提供给到其他方法和对象使用。
  3. 将回调函数onSubmit的参数类型指定为FormData。
  4. 将表单字段通过register方法，注册到react-hook-form中。

表单验证 form validate

通过register方法的第二个参数registerOptions对象可以设置一些基础的验证规则和验证方法。

示例

 1// src/components/base-form.tsx
 2import { FC } form 'react'
 3import { useForm } from 'react-hook-form'
 4
 5const BaseForm: FC = () => {
 6  // 从hooks中解构出对应的方法
 7  const { 
 8    register, 
 9    handleSubmit,
10    formState: { errors }
11  } = useForm<FormData>()
12  
13  const onSubmit = data => console.log('form data: ', data)
14  
15  return (
16  	<div>
17    	<h1>Base Form</h1>
18      <form onSubmit={handleSubmit(onSubmit)} noValidate>
19      	<div className='form-control'>
20        	<label htmlfor='username'>Username</label>
21          <input type='text' id='username' {...register('username', {
22              required: 'username is required'
23            })} />
24          <p class='error'>{errors.username?.message}</p>
25        </div>
26        
27        <div className='form-control'>
28        	<label htmlfor='email'>Email</label>
29          <input type='text' id='email' {...register('email', {
30              required: {
31                value: true,
32                message: 'email is required'
33              },
34              pattern: {
35                value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z0-9]{2,4}$/i,
36                message: 'Invalid email address'
37              }
38            })} />
39          <p class='error'>{errors.email?.message}</p>
40        </div>
41        
42        <div className='form-control'>
43        	<label htmlfor='channel'>Channel</label>
44          <input type='text' id='channel' {...register('channel')} />
45        </div>
46        
47        <button type='submit'>Submit</button>
48      </form>
49    </div>
50  )
51}
52
53export default BaseForm

上述代码中，通过register的第二个参数对象，设置了

  • required：是否必填，该属性有两种类型的值，string或一个包含value和message的对象。
  • pattern：正则表达式，通过自定义正则表达式来检测字段值。

在useForm中解构出了formState对象中的errors，该对象包含所有错误字段的消息。当出现验证错误的时候可以通过errors.字段名称.message拿到对应的错误消息，如：errors.username?.message因为字段不一定存在错误，所以需要使用?来限制当字段为空时undefined的错误问题。

自定义验证规则 custom validate rule

在registerOptions对象中，可以通过validate属性来添加一个方法或对象中添加多个方法来自定义验证规则。可以传递回调函数作为要验证的参数，也可以传递回调函数的对象来验证所有回调函数。此函数将自行执行，而不依赖于属性中包含的 required 其他验证规则。

示例：

给email添加两个限制：

  1. 填写的email不能为admin@example.com
  2. 填写的email不能使用abc.com作为结尾
 1...
 2
 3const BaseForm = () => {
 4  ...
 5  
 6  return (
 7  	...
 8    <div className='form-control'>
 9      <label htmlFor='email'>Email</label>
10      <input
11        type='text'
12        id='email'
13        {...register('email', {
14          ...
15          validate: {
16            notAdmin: value => {
17              return value !== 'admin@example.com' || 'Enter a different email address'
18            },
19            notBlacklisted: value => {
20              return (
21                !value.endsWith('abc.com') || 'You are not allowed to use this domain'
22              )
23            }
24          }
25        })}
26        />
27      <p className='error'>{errors.email?.message}</p>
28    </div>
29    ...
30  )
31}
32
33export default BaseForm

上述代码中，给validate添加了两个自定义方法noAdmin和notBlacklisted来满足前面的验证要求。

异步验证 async validate

通常会通过向服务器请求，验证用户名或邮箱是否重复，就会存在异步验证的情况。这种场景，也可以通过在validate中添加异步请求来对字段值进行验证。

示例

验证email是否为jsonplaceholder这个站点中存在的邮箱

 1...
 2
 3const BaseForm = () => {
 4  ...
 5  
 6  return (
 7  	...
 8    <div className='form-control'>
 9      <label htmlFor='email'>Email</label>
10      <input
11        type='text'
12        id='email'
13        {...register('email', {
14          ...
15          validate: {
16            ...
17            emailExists: async fieldValue => {
18              const response = await fetch(
19                `https://jsonplaceholder.typicode.com/users?email=${fieldValue}`
20              )
21              const data = await response.json()
22              return data.length === 0 || 'Email already exists'
23            }
24          }
25        })}
26        />
27      <p className='error'>{errors.email?.message}</p>
28    </div>
29    ...
30  )
31}
32
33export default BaseForm

上述代码中，向jsonplaceholder进行邮箱查询，如果返回的数据长度不为0，则表示查询到了存在的邮箱，返回错误消息。

常用配置属性 registerOptions

对于register这个方法有一些常用的配置属性（registerOptions）：

名称描述
required `string{ value: boolean, message: string }`
maxLength { value: number, message: string }输入值的最大长度，如：密码长度，最大为24个字符
minLength { value: number, message: string }输入值的最小长度，如：密码长度，最小为6个字符
max { value: number, message: string }输入值的最大值，如：年龄最大为99岁
min { value: number, message: string }输入值的最小值，如：年龄最小为1岁

validate `FunctionObject`

其他一些参数在后续进行说明。

数值和日期类型的值 numeic and date

从表单字段中获取的值一般为字符串，在onSubmit中打印时，可以看到所有的数据都是字符串。但在某些场景中，需要获取数值和日期类型的数据值。这时就需要使用到之前在registerOptions中未提到的几个属性，valueAsNumber、valueAsDate。

名称类型描述

valueAsDateboolean返回一个日期对象。 如出现问题，将返回无效日期。1. 转换发生在验证之前。2. 仅支持input标签。 3. 不会转换默认值。

监视字段值 watch field value

在react-hook-form中使用watch来监视指定的输入并返回其值。可用于呈现输入值和确定要按条件呈现的内容。

1watch: （names?： string | string[] |（data， options） => void） => unknown

示例：

 1// src/components/form.tsx
 2const { useEffect } from 'react'
 3...
 4const BaseForm = () => {
 5  const { register, handleSubmit, formState, watch } = useForm<FormData>({
 6    defaultValues: {
 7      username: '',
 8      email: '',
 9      channel: '',
10      social: {
11        twitter: '',
12        facebook: ''
13      }
14    }
15  })
16  
17  // 监视表单所有字段
18  const watchForm = watch()
19  
20  // 监视表单单个字段
21  const watchField = watch('username')
22  
23  // 监视表单多个字段
24  const watchFields = watch(['username', 'email'])
25  
26  // 使用useEffect来控制直接使用watch造成的多次渲染的问题
27  useEffect(() => {
28    const subscription = watch((value) => {
29      console.log('watch form all fields: ', value)
30    })
31    
32    return () => subscription.unsubscribe()
33  }, [watch])
34  
35  const onSubmit = data => {
36    console.log('form data: ', data)
37  }
38  
39  return (
40  	<div>
41      <p>watch form: {JSON.stringify(watchForm)}</p>
42      <p>watch field: {watchField}</p>
43      <p>watch fields: {watchFields}</p>
44      
45    	<form onSubmit={handleSubmit(onSubmit)} noValidate>
46      	...
47      </form>
48    </div>
49  )
50}
51
52export default BaseForm

上述代码中，使用了从register中解构出来的watch来对表单或表单中的字段进行监视，但需要注意，直接在组件中使用watch会造成组件的多次渲染，需要结合useEffect来做相应的监视，能避免多次渲染的问题，但仍需要对表单做防抖处理。

获取表单值 getValues

getValues用于读取表单值的优化帮助程序。和 getValues 之间的区别 watch 在于， getValues 它不会触发重新渲染或订阅输入更改。

1getValues: (payload?: string | string[]) => Object

17    console.log('get form field: ', getValues('username'))
18  }

23  ...

25  	<div>
26    	<form onSubmit={handleSubmit(onSubmit)} noValidate>
27      	...

29        <button type='button' onClick={handleGetFormValue}>Get Form Value</button>

交互和修改状态 touched and dirty

验证模式 validate mode

验证模式是指在什么交互场景来触发验证。react-hook-form提供了几种模式：onChange、onBlur、onSubmit、onTouched、all

1mode: onChange | onBlur | onSubmit | onTouched | all = 'onSubmit'
名称类型描述
onChangestring在表单项进行修改时触发验证，这种方式会触发重新渲染，对性能有影响。
onBlurstring在表单项失焦时触发验证。
onSubmitstring在提交时触发验证。
onTouchedstring是onBlur和onChange的结合，初次会在失焦后触发，后面每次会在修改时进触发验证。
allstring和onTouched类似，但每次失焦、每次修改都会触发验证。

13  })

38      email: '',

40    },

43
44  const onSubmit = (data: FormData) => {
