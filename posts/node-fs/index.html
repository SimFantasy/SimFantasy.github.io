Node 02 Fs 模块

Simz

Node

fs是File System的缩写，表示文件系统。node:fs 模块能够以标准 POSIX 函数为模型的方式与文件系统进行交互。

所有文件系统操作都具有同步、回调和基于 promise 的形式，并且可以使用 CommonJS 语法和 ES6 模块进行访问。

  1. 同步操作文件，代码会被阻塞，当文件操作代码执行完成，从会继续执行后面的代码。
  2. 异步回调函数操作文件，代码不会被阻塞，需要传入回调函数，当获取到结果时，回调函数被执行。
  3. 异步Pormise操作文件，代码不会被阻塞，通过fs.promises调用方法操作，会返回一个promise，可以通过then、catch进行处理。

以下fs模块操作api介绍，仅异步回调方式，同步及promise异步方式见官方文档。

文件操作

读取文件 fs.readFile(path[, options], callback)

  • path <string> | <Buffer> | <URL> | <integer> 文件名或文件描述符
  • options <Object> | <string>
    • encoding | 默认值： null
    • flag 参见文件系统 flags 的支持。 默认值： ‘r’。
    • signal 允许中止正在进行的读取文件
  • callback <Function>
    • err |
    • data |

异步地读取文件的全部内容。

示例：

1const { readFile } = require('fs')
2
3readFile('./test.txt', { encoding: 'utf8' }, (err, data) => {
4	if (err) return console.log(err)
5	console.log(data)
6})
7// hello world ~

文件描述符（File descriptiors）

在 POSIX 系统上，对于每个进程，内核维护一个当前打开的文件和资源表。 每个打开的文件都分配有一个简单的数字标识符，称为文件描述符。 在系统级，所有文件系统操作都使用这些文件描述符来识别和跟踪每个特定文件。 Windows 系统使用不同但概念上相似的机制来跟踪资源。 为了方便用户，Node.js 抽象了操作系统之间的差异，并为所有打开的文件分配了一个数字文件描述符。

基于回调的 fs.open() 和同步 fs.openSync() 方法打开一个文件并分配一个新的文件描述符。 分配后，文件描述符可用于从文件读取数据、向文件写入数据或请求有关文件的信息。

操作系统限制在任何给定时间可能打开的文件描述符的数量，因此在操作完成时关闭描述符至关重要。关闭使用fs.close(fd) ，否则将导致内存泄漏，最终导致应用崩溃。

示例：

 1const { open, close, fstat } = require('fs')
 2
 3open('./test.txt', (err, fd) => {
 4	if (err) {
 5		console.log(err)
 6		return
 7	}
 8
 9	console.log('文件打开成功')
10
11	// 获取文件描述符
12	console.log(fd)
13
14	// 通过文件描述符获取文件信息
15	// fstat 使用文件描述符的 <fs.Stats> 调用回调。
16	fstat(fd, (err, stats) => {
17		if (err) {
18			console.log(err)
19			return
20		}
21		console.log(stats.size)
22
23		// 关闭文件
24		close(fd, err => {
25			if (err) {
26				console.log(err)
27				return
28			}
29			console.log('文件关闭成功')
30		})
31	})
32})

打开文件 fs.open(path[, flags[, mode]], callback)

  • path <string> | <Buffer> | <URL>
  • flags <string> | <number> 参见文件系统 flags 的支持。 默认值： 'r'。
  • mode <string> | <integer> 默认值： 0o666（可读写）
  • callback <Function>
    • err
    • fd

异步地打开文件。

写入文件 fs.writeFile(file, data[, options], callback)

  • file <string> | <Buffer> | <URL> | <integer> 文件名或文件描述符
  • data <string> | <Buffer> | <TypedArray> | <DataView> | <Object>
  • options <Object> | <string>
    • encoding | 默认值： ‘utf8’
    • mode 默认值： 0o666
    • flag 默认值： ‘w’。
    • signal 允许中止正在进行的写入文件
  • callback <Function>
    • err |

当 file 是文件名时，将数据异步地写入文件，如果文件已存在则替换该文件。 data 可以是字符串或缓冲区。

当 file 是文件描述符时，其行为类似于直接调用 fs.write()（推荐）。

如果 data 是缓冲区，则忽略 encoding 选项。

mode 选项仅影响新创建的文件。

示例：

 1const { writeFile } = require('fs')
 2
 3// 写入文件内容
 4const content = 'hello node.js'
 5
 6// 写入文件
 7writeFile('./test.txt', content, { flag: 'a+' }, err => {
 8	if (err) {
 9		console.log('写入文件失败', err)
10		return
11	}
12	console.log('写入文件成功')
13})

Flag 文件系统标志

以下标志在 flag 选项接受字符串的任何地方可用。一般使用'a+'来对文件内容进行追加，而不会使用默认值对文件进行内容覆盖。

  • 'a': 打开文件进行追加。 如果文件不存在，则创建该文件。

  • 'ax': 类似于 'a' 但如果路径存在则失败。

  • 'a+': 打开文件进行读取和追加。 如果文件不存在，则创建该文件。

  • 'ax+': 类似于 'a+' 但如果路径存在则失败。

  • 'as': 以同步模式打开文件进行追加。 如果文件不存在，则创建该文件。

  • 'as+': 以同步模式打开文件进行读取和追加。 如果文件不存在，则创建该文件。

  • 'r': 打开文件进行读取。 如果文件不存在，则会发生异常。

  • 'r+': 打开文件进行读写。 如果文件不存在，则会发生异常。

  • 'rs+': 以同步模式打开文件进行读写。 指示操作系统绕过本地文件系统缓存。

    这主要用于在 NFS 挂载上打开文件，因为它允许跳过可能过时的本地缓存。 它对 I/O 性能有非常实际的影响，因此除非需要，否则不建议使用此标志。

    这不会将 fs.open() 或 fsPromises.open() 变成同步阻塞调用。 如果需要同步操作，应该使用类似 fs.openSync() 的东西。

  • 'w': 打开文件进行写入。 创建（如果它不存在）或截断（如果它存在）该文件。

  • 'wx': 类似于 'w' 但如果路径存在则失败。

  • 'w+': 打开文件进行读写。 创建（如果它不存在）或截断（如果它存在）该文件。

  • 'wx+': 类似于 'w+' 但如果路径存在则失败。

目录（文件夹）操作

创建目录 fs.mkdir(path[, options], callback)

  • path <string> | <Buffer> | <URL>
  • options <Object> | <integer>
    • recursive 递归，默认值： false
    • mode | Windows 上不支持。 默认值： 0o777。
  • callback <Function>
    • err
    • path | 仅当创建目录时将 recursive 设置为 true。

异步地创建目录。

示例：

 1const { mkdir } = require('fs')
 2
 3// 创建一级目录
 4mkdir('./demo', err => {
 5	if (err) {
 6		console.log(err)
 7		return
 8	}
 9	console.log('创建成功')
10})
11
12const { mkdir } = require('fs')
13
14// 创建多级目录，使用recursive来进行递归创建多级目录
15mkdir('./demo0/demo1/demo2', { recursive: true }, err => {
16	if (err) {
17		console.log(err)
18		return
19	}
20	console.log('创建成功')
21})

读取目录 fs.readdir(path[, options], callback)

  • path <string> | <Buffer> | <URL>
  • options <string> | <Object>
    • encoding 默认值： ‘utf8’
    • withFileTypes 默认值： false
  • callback <Function>
    • err
    • files <string[]> | <Buffer[]> | <fs.Dirent[]>

读取目录的内容。 回调有两个参数 (err, files)，其中 files 是目录中文件名的数组，不包括 '.' 和 '..'。

可选的 options 参数可以是指定编码的字符串，也可以是具有 encoding 属性（指定用于传给回调的文件名的字符编码）的对象。 如果 encoding 设置为 'buffer'，则返回的文件名将作为 对象传入。

如果 options.withFileTypes 设置为 true，则 files 数组将包含 <fs.Dirent> 对象。(目录条目的表示，可以是目录中的文件或子目录，通过从 <fs.Dir> 读取返回。 目录条目是文件名和文件类型对的组合。)

示例：

 1const { readdir } = require('fs')
 2
 3// 读取目录，但此方式不能读取到文件类型，不知道是否为文件夹或文件（可能存在没有后缀的文件）
 4readdir('./demo', (err, files) => {
 5	if (err) {
 6		console.log(err)
 7	} else {
 8		console.log(files) // [ 'demo1', 'foo.txt' ]
 9	}
10})
11
12// 读取目录，添加参数withFileTypes，可以获取到文件类型，[Symbol(type)]: 2为目录，[Symbol(type)]: 1为文件
13readdir('./demo', { withFileTypes: true }, (err, files) => {
14	if (err) {
15		console.log(err)
16	} else {
17		console.log(files) 
18	}
19})
20
21// [
22//   Dirent { name: 'demo1', [Symbol(type)]: 2 },
23//   Dirent { name: 'foo.txt', [Symbol(type)]: 1 }
24// ]

通过递归的方式读取多级目录下的文件：

 1const { readdir } = require('fs')
 2
 3// 通过递归读取目录下的所有文件
 4function readDirFn(path) {
 5	readdir(path, { withFileTypes: true }, (err, files) => {
 6		if (err) {
 7			console.log(err)
 8			return
 9		}
10		files.forEach(file => {
11			if (file.isDirectory()) {
12				readDirFn(`${path}/${file.name}`)
13			} else {
14				console.log(file.name)
15			}
16		})
17	})
18}
19readDirFn('./demo')
20
21// foo.txt
22// bar.txt
23// baz.txt

重命名文件或目录 fs.rename(oldPath, newPath, callback)

  • oldPath <string> | <Buffer> | <URL>
  • newPath <string> | <Buffer> | <URL>
  • callback <Function>
    • err

将 oldPath 处的文件或目录异步重命名为作为 newPath 提供的文件名称或目录名称。 如果 newPath 已经存在，则它将被覆盖。 如果 newPath 是目录，则会引发错误。

示例：

 1const { rename } = require('fs')
 2
 3// 重命名文件夹
 4rename('./demo', './demoOne', err => {
 5	if (err) throw err
 6	console.log('重命名成功')
 7})
 8
 9// 重命名文件
10rename('./test.txt', './demo.txt', err => {
11	if (err) throw err
12	console.log('重命名成功')
13})

删除目录 fs.rmdir(path[, options], callback)

  • path <string> | <Buffer> | <URL>
  • options <Object>
    • maxRetries 如果遇到 EBUSY、EMFILE、ENFILE、ENOTEMPTY 或 EPERM 错误，Node.js 将在每次尝试时以 retryDelay 毫秒的线性退避等待时间重试该操作。 此选项表示重试次数。 如果 recursive 选项不为 true，则忽略此选项。 默认值： 0。
    • recursive 如果为 true，则执行递归目录删除。 在递归模式下，操作将在失败时重试。 默认值： false。 Deprecated.
    • retryDelay 重试之间等待的时间（以毫秒为单位）。 如果 recursive 选项不为 true，则忽略此选项。 默认值： 100。
  • callback <Function>
    • err

异步地删除空目录。目录中不能存在文件，否则报错。

在文件（而不是目录）上使用 fs.rmdir()，则在 Windows 上会导致 ENOENT 错误，在 POSIX 上会导致 ENOTDIR 错误。

示例：

 1// 删除空目录
 2rmdir('./test', err => {
 3	if (err) {
 4		console.log(err)
 5		return
 6	}
 7	console.log('目录删除成功')
 8})
 9
10// 删除多级空目录
11rmdir('./test', { recursive: true }, err => {
12	if (err) {
13		console.log(err)
14		return
15	}
16	console.log('删除目录成功')
17})

删除目录或文件 fs.rm(path[, options], callback)

  • path <string> | <Buffer> | <URL>
  • options <Object>
    • force 当为 true 时，如果 path 不存在，则异常将被忽略。 默认值： false。
    • maxRetries 如果遇到 EBUSY、EMFILE、ENFILE、ENOTEMPTY 或 EPERM 错误，Node.js 将在每次尝试时以 retryDelay 毫秒的线性退避等待时间重试该操作。 此选项表示重试次数。 如果 recursive 选项不为 true，则忽略此选项。 默认值： 0。
    • recursive 如果为 true，则执行递归删除。 在递归模式下，操作将在失败时重试。 默认值： false。
    • retryDelay 重试之间等待的时间（以毫秒为单位）。 如果 recursive 选项不为 true，则忽略此选项。 默认值： 100。
  • callback <Function>
    • err

异步地删除文件和目录。

要获得类似于 rm -rf Unix 命令的行为，则使用具有选项 { recursive: true, force: true }

示例：

1const { rm } = require('fs')
2
3// 删除目录，目录下有文件，此操作，会将目录下所有文件或目录进行删除，不管目录层级
4rm('./demoOne', { recursive: true, force: true }, err => {
5	if (err) return console.log(err)
6	console.log('modules目录已删除')
7})
