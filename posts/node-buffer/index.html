Node 04 Buffer 类

Simz
2023-10-08
Node

计算机中所有的内容：文字、数字、图片、音频、视频最终都会使用二进制来表示。

服务器要处理的本地文件类型相对较多，比如某一个保存文本的文件并不是使用 utf-8进行编码的，而是用 GBK，那么我们必须读取到他们的二进制数据，再通过GKB转换成对应的文字。对于服务器端为了做很多的功能，我们必须直接去操作其二进制的数据。所以Node为了可以方便开发者完成更多功能，提供给了我们一个类Buffer，并且它是全局的。

在服务器端都需要现将本地文件转为Buffer的二进制格式，然后处理完成后再写入到对应的文件中。

Buffer中存储的是二进制数据，如何存储：

  • 可以将Buffer看成是一个存储二进制的数组
  • 这个数组中的每一项，可以保存8位二进制： 00000000
  • Buffer相当于是一个字节的数组，数组中的每一项对应一个字节的大小

Buffer.from(string[, encoding])

  • string 要编码的字符串。
  • encoding string 的编码。 默认值： 'utf8'。

创建包含 string 的新 Buffer。 encoding 参数标识将 string 转换为字节时要使用的字符编码。

示例：

 1// 使用Buffer.from方法将文本转成Buffer
 2const buf1 = Buffer.from('hello world')
 3console.log(buf1) // <Buffer 68 65 6c 6c 6f 20 77 6f 72 6c 64>
 4
 5// 指定编码类型为base64
 6const buf2 = Buffer.from('hello world', 'utf16le')
 7console.log(buf2) // <Buffer 68 00 65 00 6c 00 6c 00 6f 00 20 00 77 00 6f 00 72 00 6c 00 64 00>
 8
 9// 通过toString方法将buffer转成文本，但是需要对应的编码
10const bufStr = buf1.toString()
11console.log(bufStr) // hello world
12
13const buf2Str = buf2.toString('utf16le')
14console.log(buf2Str) // hello world

Buffer.alloc(size[, fill[, encoding]])

  • size 新的 Buffer 所需的长度。
  • fill | | | 用于预填充新 Buffer 的值。 默认值： 0。
  • encoding 如果 fill 是字符串，则这就是它的编码。 默认值： 'utf8'。

分配 size 个字节的新 Buffer。 如果 fill 为 undefined，则 Buffer 将被填零。

示例：

 1// 不指定填充内容，默认填充为0
 2const buf01 = Buffer.alloc(10)
 3console.log(buf01) // <Buffer 00 00 00 00 00 00 00 00 00 00>
 4
 5// 指定填充内容a
 6const buf02 = Buffer.alloc(10, 'a')
 7console.log(buf02) // <Buffer 61 61 61 61 61 61 61 61 61 61>
 8
 9// 指定填充内容a和编码为utf16le
10const buf03 = Buffer.alloc(10, 'a', 'utf16le')
11console.log(buf03) // <Buffer 61 00 61 00 61 00 61 00 61 00>

buffer可以通过索引对单个字节来操作获得对应的值

示例：

 1const buf01 = Buffer.alloc(10)
 2console.log(buf01) // <Buffer 00 00 00 00 00 00 00 00 00 00>
 3
 4// 通过索引，来查看 buf01 中的字节值
 5console.log('buf01[0]:', buf01[0])
 6console.log('buf01[1]:', buf01[1])
 7
 8// 通过索引，来修改 buf01 中的字节值
 9buf01[0] = 100
10buf01[1] = 0x66
11buf01[2] = 'm'.charCodeAt() // 通过charCodeAt()方法将字母转成字符的 ASCII 码来修改
12
13console.log(buf01) // <Buffer 64 66 6d 00 00 00 00 00 00 00>
14
15// 查看修改的字节值
16console.log(buf01.toString()) // dfm

buffer的高级使用，通过修改buffer字节来修改对应文件中的内容。

 1const fs = require('fs')
 2
 3fs.readFile('./demo.txt', (err, data) => {
 4	if (err) {
 5		console.log('err', err)
 6		return
 7	}
 8	console.log('data', data)
 9	// 修改data中的字节内容，将world修改为node
10	data[6] = 'n'.charCodeAt()
11	data[7] = 'o'.charCodeAt()
12	data[8] = 'd'.charCodeAt()
13	data[9] = 'e'.charCodeAt()
14	data[10] = 0x0a // 换行符
15	console.log('data', data)
16	console.log('data:', data.toString())
17
18	// 写入文件
19	fs.writeFile('./demo.txt', data, err => {
20		if (err) {
21			console.log('err', err)
22			return
23		}
24		console.log('写入成功')
25	})
26})
