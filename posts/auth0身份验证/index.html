Auth0身份验证

Simz
2022-10-27
react auth0

Auth0是一个第三方身份验证的工具。通过配置可以调用Auth0的注册、登录，配置指定回调地址就可以很快实现身份验证的功能。

‍

注册、设置Auth0

  1. 基础部分很简单，首先到Auth0注册一个账号，直接用Github或者Google的账号就可以直接登录。

  2. 然后在进入到Dashboard以后在左侧栏找到Application应用模块下的应用列表。

    ​

  3. 然后添加应用。添加应用的名字，选择使用的应用类型，根据下面的说明 我们选择第二个单页网页应用。

  4. 添加成功后，在应用面板中第二个标签页settings中可以看到Auth0的基础信息，包含域名Domain和客户IDClient ID，这两个是需要在项目中添加登录时配置的。

  5. 然后往下，在Application URIs中可以看到应用登录的登录端点、回调链接、注销网址等，我们在这里配置下允许的回调网址Allowed Callback URLs和运行注销网址。因为会用到vite和next端口不一致，可以使用英文逗号进行分割。填写完成保存修改。

  6. 然后在上面第四个标签页连接Connections中可以看到可以使用的链接方式，和已经添加的第三方链接信息。这里可以控制连接源。

  7. 在左侧的菜单栏中第四个模块认证Authentication中第一个是配置默认登录，一般为使用邮箱登录。第二个是社交Social，可以选择第三方的登录，里面的内容很多可以仔细研究下。

  8. 这里添加好第三方社交的连接，需要到应用中开启下，否则默认是关闭的。

‍

项目使用

安装

1 pnpm add @auth0/auth0-react

以下以vite项目为例。

‍

Main

安装好后在入口文件中配置Auth0的Provider。但首先我们把之前的Domain和Client ID配置到.env文件中。在入口文件中进行引用。

 1// src/main.js
 2
 3import React from 'react'
 4import ReactDOM from 'react-dom/client'
 5import App from '@/App'
 6import { Auth0Provider } from '@auth0/auth0-react'
 7import '@/index.css'
 8
 9const domain = import.meta.env.VITE_AUTH0_DOMAIN
10const clientId = import.meta.env.VITE_AUTH0_CLIENT_ID
11
12ReactDOM.createRoot(document.getElementById('root')).render(
13  <Auth0Provider domain={domain} clientId={clientId} redirectUri={window.location.origin}>
14    <App />
15  </Auth0Provider>

redirectUri：是在Auth0在身份验证通过后将用户重新定向到应用程序的URL。可以在之前配置中找到对应的字段配置值。window.location.origin是默认原来的页面。

‍

App

Auth0的登录很简单，就是一个登录按钮，点击跳转到他们的登录页面进行登录，成功了就再跳转回来。

先

 1// src/App.js
 2
 3import { useAuth0 } from "@auth0/auth0-react";
 4import React from "react";
 5
 6const App = () => {
 7  const { user, isAuthenticated, isLoading } = useAuth0();
 8
 9  if (isLoading) {
10    return <div>Loading ...</div>;
11  }
12
13  return (
14    isAuthenticated && (
15      <div>
16        <LoginButton />
17        <LogoutButton />
18        <Profile />
19      </div>
20    )
21  );
22};
23
24export default App;

‍

Login

在登录组件中需要做好验证，如果没验证显示。

 1// src/components/Login.jsx
 2
 3import React from 'react'
 4import { useAuth0 } from '@auth0/auth0-react'
 5
 6const Login = () => {
 7  const { loginWithRedirect, isAuthenticated } = useAuth0()
 8  return (
 9    !isAuthenticated && (
10      <button onClick={() => loginWithRedirect()}>
11        登录
12      </button>
13    )
14  )
15}
16
17export default Login

‍

Logout

也是在组件中作好验证，如果验证通过显示。

 1// src/components/Logout.jsx
 2
 3import React from 'react'
 4import { useAuth0 } from '@auth0/auth0-react'
 5
 6const LogoutButton = () => {
 7  const { logout, isAuthenticated } = useAuth0()
 8  return (
 9    isAuthenticated && (
10      <button onClick={() => logout()}>
11        登出
12      </button>
13    )
14  )
15}
16
17export default LogoutButton

‍

Profile

登录后可以通过user获取到用户信息，在Auth0的配置中可以配置拿到什么信息，进行什么操作的关联。

 1// src/components/Profile.jsx
 2
 3import React from 'react'
 4import { useAuth0 } from '@auth0/auth0-react'
 5
 6const Profile = () => {
 7  const { user, isAuthenticated } = useAuth0()
 8  return (
 9    isAuthenticated && (
10      <article>
11        <div>
12          <div>
13            <img src={user.picture} />
14          </div>
15          <div>
16            <div>{user.name}</div>
17            <div>{user.nickname}</div>
18          </div>
19        </div>
20      </article>
21    )
22  )
23}
24
25export default Profile

总结

以上简单几步就完成了身份验证的功能，并且还可以通过Auth0对验证进行很强大的配置。之前有用到Google的验证，麻烦的一批，最后验证还不能用同一个谷歌账号，只好作罢。Auth0也让我看到一个简单的验证可以做的这么简单且强大。
